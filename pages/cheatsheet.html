3 · Go templates you must be able to type from memory

Sliding window

l := 0; best := 0; freq := map[byte]int{}
for r := 0; r < len(s); r++ {
    freq[s[r]]++
    for /* invalid */ {
        freq[s[l]]--; if freq[s[l]] == 0 { delete(freq, s[l]) }
        l++
    }
    if r-l+1 > best { best = r - l + 1 }
}
return best

Binary search (lower-bound idiom)

lo, hi := 0, len(a) // search in [lo, hi)
for lo < hi {
    mid := lo + (hi-lo)/2
    if a[mid] < target { lo = mid + 1 } else { hi = mid }
}
// lo == first index with a[lo] >= target (or len(a))

BFS (graph / grid)

q := []int{src}
seen := map[int]bool{src: true}
for len(q) > 0 {
    u := q[0]; q = q[1:]
    for _, v := range adj[u] {
        if seen[v] { continue }
        seen[v] = true
        q = append(q, v)
    }
}

DFS with stack

stack := []int{src}; seen := map[int]bool{src:true}
for len(stack) > 0 {
    u := stack[len(stack)-1]; stack = stack[:len(stack)-1]
    for _, v := range adj[u] {
        if !seen[v] { seen[v]=true; stack = append(stack, v) }
    }
}

Dijkstra

dist := make([]int, n)
for i := range dist { dist[i] = math.MaxInt }
dist[src] = 0
pq := &MinHeap{{0, src}}
heap.Init(pq)
for pq.Len() > 0 {
    cur := heap.Pop(pq).(Item)
    if cur.d > dist[cur.v] { continue }
    for _, e := range adj[cur.v] {
        if nd := cur.d + e.w; nd < dist[e.to] {
            dist[e.to] = nd
            heap.Push(pq, Item{nd, e.to})
        }
    }
}

Union-Find

type DSU struct{ p, r []int }
func NewDSU(n int) *DSU { p := make([]int, n); for i := range p { p[i]=i }; return &DSU{p, make([]int, n)} }
func (d *DSU) Find(x int) int { if d.p[x] != x { d.p[x] = d.Find(d.p[x]) }; return d.p[x] }
func (d *DSU) Union(a, b int) bool {
    a, b = d.Find(a), d.Find(b)
    if a == b { return false }
    if d.r[a] < d.r[b] { a, b = b, a }
    d.p[b] = a
    if d.r[a] == d.r[b] { d.r[a]++ }
    return true
}

Top-K with min-heap

h := &MinHeap{}; heap.Init(h)
for _, x := range nums {
    heap.Push(h, x)
    if h.Len() > k { heap.Pop(h) }
}
// h now holds the k largest; root is the k-th largest

4 · Go gotchas you must NOT trip on

  • Loop variable capture (pre-1.22): closures over for i,v := range share i,v. Shadow with i,v := i,v.
  • Append aliasing: if cap > len, append mutates the shared backing array. Reassign always.
  • Nil interface ≠ typed nil pointer. var p *T = nil; var i I = p; i == nil // false.
  • Map writes are not safe — concurrent map writes panic. Use sync.Map or a Mutex.
  • Range over map is unordered — randomized per iteration.
  • Channels: send on closed → panic. Receive on closed → zero, ok=false. Close once, by sender.
  • defer evaluates args at defer-time; the call runs at return.
  • Always pass context.Context as the first arg after the receiver.
  • Errors: wrap with %w, compare with errors.Is/errors.As. Don't return generic "error" strings.
  • time.After leaks in selects until the timer fires — use time.NewTimer + Stop in hot paths.

5 · Concurrency one-liners

  • WaitGroup: wg.Add(1); go func(){ defer wg.Done(); ... }(); wg.Wait().
  • Errgroup: g, ctx := errgroup.WithContext(ctx); g.Go(...); err := g.Wait().
  • Worker pool: jobs chan in, results chan out, N goroutines, close jobs to stop.
  • Timeout: ctx, cancel := context.WithTimeout(parent, 2*time.Second); defer cancel().
  • Single-flight: singleflight.Group dedupes concurrent identical fetches.
  • Rate limit: rate.NewLimiter(rate.Limit(qps), burst) from x/time/rate.

7 · The 7-step system-design framework

  1. Requirements — functional + non-functional (latency, availability, durability, consistency, scale).
  2. Capacity — QPS, storage, bandwidth, working set.
  3. API — REST/gRPC, idempotency keys, pagination.
  4. Architecture — client → LB → service → cache → DB → async pipeline.
  5. Data model — schemas, indexes, DB choice, sharding key.
  6. Deep dives — pick 2–3 components, stress-test (hotspots, failure modes, observability).
  7. Trade-offs & follow-ups — 100× scale? Cross-region? Cost?

8 · Distributed-systems vocabulary you'll be asked

  • CAP — under partition, choose Consistency or Availability. PACELC — even without partition, choose Latency or Consistency.
  • Quorum: R+W > N gives strong consistency on Dynamo-style systems.
  • Consistent hashing with virtual nodes — only K/N keys move when adding/removing a node.
  • Idempotency — client supplies a key, server stores (key → result) with TTL.
  • Outbox — atomic DB write + relay → broker for at-least-once events without 2PC.
  • Saga — local txns + compensations; orchestrated (Temporal) or choreographed (events).
  • CQRS / event sourcing — split read/write models; state = fold(events).
  • Backpressure — bounded queues; slow producers when consumers lag.
  • Bulkhead — isolate resource pools so one failure can't drown the whole service.
  • Circuit breaker — fail fast for a window after repeated failures; half-open to probe.

9 · Cache patterns and invalidation

  • Cache-aside — app reads cache, misses → DB → fill cache. Simple, default choice.
  • Read-through — cache library handles miss/load. Encapsulated.
  • Write-through — write to cache + DB synchronously. Strong consistency, slower writes.
  • Write-back — write to cache, flush async. Risk of data loss on crash.
  • Stampede protection — singleflight, request coalescing, jittered TTLs, pre-warm.
  • Eviction — LRU default; LFU for hot-key heavy; TTL for time-bounded data.

10 · Rate limiting algorithms (one-liners)

  • Token bucket — refill r/s up to capacity B; allow if token available. Permits bursts.
  • Leaky bucket — fixed drain rate; smooths output but no bursts.
  • Fixed window — counter per minute. Simple but allows 2× burst at boundary.
  • Sliding window log — exact, O(N) memory per key.
  • Sliding window counter — interpolation between two fixed windows. Good balance.
  • Distributed → Redis with Lua script for atomicity.

11 · Database one-liners

  • SQL — relations, transactions, joins, schema. Use first; only leave if you must.
  • Document (Mongo) — flexible schema, embedded docs, weaker txns.
  • KV (Redis, Dynamo) — O(1) by key, no joins; pick partition key carefully.
  • Wide column (Cassandra, ScyllaDB) — write-heavy, tunable consistency, time-series.
  • Search (Elastic, OpenSearch) — inverted index; near-real-time, not source of truth.
  • Graph (Neo4j) — many-hop traversals; relationships are first-class.
  • Indexes — speed reads, slow writes; B-tree default; covering index avoids row lookup.
  • Isolation — Read committed → Repeatable read → Snapshot → Serializable.
