Hash Maps & Sets

The most overused — and rightly so — tool in interviews. Hash to remember, hash to group, hash to deduplicate, hash to prefix-sum.

Mental model

  • Hashing trades space for time: O(1) average lookup, insert, delete. The cost is unordered iteration and worst-case O(n) on adversarial input.
  • Six canonical uses: lookup-by-complement, prefix-state counter, canonical signature, frequency map, set-of-seen, map+DLL (LRU/LFU).
  • Composite keys: in Go, structs and arrays are hashable; slices and maps are not. [26]int is a beautiful hashable anagram fingerprint.
  • Sets are map[T]struct{} — zero-byte values, real semantics.
  • If iteration order matters, sort the keys after the fact, or use a separate slice in insertion order.

Go map essentials

m := make(map[string]int, 64)        // size hint avoids rehashing
v, ok := m["k"]                       // comma-ok
delete(m, "k")
// Set = map[T]struct{} — zero-byte value.
set := make(map[int]struct{})
set[42] = struct{}{}
_, exists := set[42]

Two Sum (one pass)

func twoSum(a []int, t int) []int {
    seen := map[int]int{}
    for i, v := range a {
        if j, ok := seen[t-v]; ok { return []int{j, i} }
        seen[v] = i
    }
    return nil
}

Subarray Sum Equals K (prefix + map)

func subarraySum(a []int, k int) int {
    cnt := map[int]int{0: 1}          // prefix sum 0 has occurred once (empty)
    sum, ans := 0, 0
    for _, v := range a {
        sum += v
        ans += cnt[sum-k]
        cnt[sum]++
    }
    return ans
}

Group anagrams (composite key)

func groupAnagrams(ws []string) [][]string {
    g := map[[26]int][]string{}
    for _, w := range ws {
        var k [26]int
        for i := 0; i < len(w); i++ { k[w[i]-'a']++ }
        g[k] = append(g[k], w)
    }
    out := make([][]string, 0, len(g))
    for _, v := range g { out = append(out, v) }
    return out
}

Longest Consecutive Sequence — O(n)

func longestConsecutive(a []int) int {
    set := map[int]struct{}{}
    for _, v := range a { set[v] = struct{}{} }
    best := 0
    for v := range set {
        if _, ok := set[v-1]; ok { continue }   // only start at sequence heads
        n := 1
        for { if _, ok := set[v+n]; !ok { break }; n++ }
        if n > best { best = n }
    }
    return best
}

Sliding-window count map

// Maintain count of distinct elements as you slide; delete keys when count hits 0
// so len(m) == #distinct.
m := map[byte]int{}
add := func(c byte) { m[c]++ }
del := func(c byte) { m[c]--; if m[c] == 0 { delete(m, c) } }

Hash patterns gallery

  • Lookup by complement: two-sum / four-sum II.
  • Prefix-state counts: subarray sum/divisibility/XOR equals K.
  • Canonical signature: anagrams, isomorphic strings, group shifted strings.
  • Frequency map + sort/heap: top-K frequent.
  • Set of seen states: cycle detection in linked list (alt to Floyd), happy number.
  • Map + DLL: LRU/LFU caches.
  • Map of buckets: bucket sort by frequency for O(n) top-K.

Pitfalls

  • Map iteration order is randomized — never depend on it.
  • You can't take an address of a map value (&m["k"]) — assign first.
  • Reading from nil map is fine; writing panics.
  • Hash collisions on attacker-supplied keys can DoS — Go randomizes the hash seed per process.
  • Concurrent map writes → fatal error: concurrent map writes (kills the process). Use sync.RWMutex or sync.Map when shared.
  • Forgetting to delete entries with count 0 in a sliding-window distinct-count map.

Anti-patterns — what NOT to say

  • "sync.Map for general-purpose concurrency." Niche; map + RWMutex is usually faster.
  • Storing interface{} values to avoid type-aware logic — boxing kills locality.
  • Using a map where a small fixed array would do (e.g. [26]int).
  • Sorting keys to "guarantee order" instead of using a separate insertion-order slice.
  • "Hash everything in case I need it later" — premature space cost.

Senior interviewer follow-ups

ProbeWhat they want to hear
"Two Sum without a hash map?"Sort + two pointers — O(n log n) but O(1) extra. Loses original indices.
"Why prefix-sum + map for Subarray Sum = K?"Looking up prefix - K answers "is there an earlier prefix that, removed, leaves sum K?".
"How do you bucket-sort top-K frequent in O(n)?"Frequency map → buckets indexed by frequency → walk from high to low.
"What does Go do to prevent hash-DoS?"Random hash seed per process; for adversarial keys, bucket counts are bounded.
"Insert/Delete/GetRandom O(1)?"Map of value→index in slice; delete = swap with last and pop.
"How do you know to use [26]int as a key?"Array is hashable, fixed size, fast to compare. Strings build into it via k[c-'a']++.
