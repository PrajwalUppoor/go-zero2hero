Intervals

Sort + sweep. 80% of interval problems reduce to a sort by start (or end) followed by a linear scan with a tiny state machine or a heap.

Mental model

  • Five reflexes:
    1. Sort by start → merge / insert.
    2. Sort by end → max non-overlapping (greedy).
    3. Sweep line → events (time, +1/-1) for max concurrency, skyline.
    4. Heap of end times → minimum rooms / available resources.
    5. Difference array → bulk range updates without per-update O(n).
  • Define your boundaries upfront: closed-closed [a,b], half-open [a,b), or closed-open. Touching endpoints (does [1,2] overlap [2,3]?) decide your overlap predicate.
  • For "is this point in some interval" over many queries, sort intervals + binary search (or precompute via difference array).
  • For dynamic insertions/deletions, reach for a TreeMap-equivalent — Go has no built-in, so use a balanced BST library, or maintain sorted slice with binary search insert.

Merge intervals

func merge(iv [][]int) [][]int {
    sort.Slice(iv, func(i, j int) bool { return iv[i][0] < iv[j][0] })
    var out [][]int
    for _, x := range iv {
        if len(out) > 0 && x[0] <= out[len(out)-1][1] {
            if x[1] > out[len(out)-1][1] { out[len(out)-1][1] = x[1] }
        } else {
            out = append(out, x)
        }
    }
    return out
}

Insert interval

func insert(iv [][]int, n []int) [][]int {
    var out [][]int; i := 0
    for ; i < len(iv) && iv[i][1] < n[0]; i++ { out = append(out, iv[i]) }
    for ; i < len(iv) && iv[i][0] <= n[1]; i++ {
        n[0] = min(n[0], iv[i][0]); n[1] = max(n[1], iv[i][1])
    }
    out = append(out, n)
    out = append(out, iv[i:]...)
    return out
}

Meeting Rooms II — minimum rooms

// Heap of end times; if next start >= min end, reuse a room.
func minMeetingRooms(iv [][]int) int {
    sort.Slice(iv, func(i, j int) bool { return iv[i][0] < iv[j][0] })
    h := &IntHeap{}
    for _, m := range iv {
        if h.Len() > 0 && (*h)[0] <= m[0] { heap.Pop(h) }
        heap.Push(h, m[1])
    }
    return h.Len()
}

Sweep line template

// Events: +1 at start, -1 at end (or end+1 for inclusive). Sort, walk, track running count.
events := make([][2]int, 0, 2*len(iv))
for _, x := range iv {
    events = append(events, [2]int{x[0], +1})
    events = append(events, [2]int{x[1], -1})
}
sort.Slice(events, func(i, j int) bool {
    if events[i][0] != events[j][0] { return events[i][0] < events[j][0] }
    return events[i][1] < events[j][1]            // process -1 before +1 to avoid double-counting touch
})
cur, best := 0, 0
for _, e := range events {
    cur += e[1]
    if cur > best { best = cur }
}

Non-overlapping intervals (erase minimum)

// Greedy: sort by END, keep last end, count removals.
sort.Slice(iv, func(i, j int) bool { return iv[i][1] < iv[j][1] })
end, rem := math.MinInt, 0
for _, x := range iv {
    if x[0] >= end { end = x[1] } else { rem++ }
}

Difference array (range update + final query)

// Add v to [l, r] in O(1); final pass produces actual values in O(n).
diff := make([]int, n+1)
diff[l] += v
diff[r+1] -= v
// Final
cur := 0
for i := 0; i < n; i++ { cur += diff[i]; arr[i] = cur }

Anti-patterns — what NOT to say

  • "Compare every pair." O(n²) when sorting + sweep is O(n log n).
  • Using []Interval with field comparators when a [][]int is faster.
  • Ignoring tie-breaking on sweep-line events — touching at a point gives wrong concurrency.
  • Modifying iv while iterating without resetting indices.
  • Using a heap when a simple end tracker suffices (Non-overlapping Intervals).

Senior interviewer follow-ups

ProbeWhat they want to hear
"How would you handle 10⁷ intervals streaming?"Online sweep — ingest events, track running concurrency; or maintain a balanced BST of merged intervals.
"Meeting Rooms II without a heap?"Sweep line: sort starts and ends separately; walk both pointers; rooms = max(rooms, active).
"How does the sweep line ordering matter?"For "max concurrent", process ends before starts at the same time (interval ends before another starts at exact same instant).
"Skyline — why max-heap with lazy deletion?"We need the current max active height; lazily skip popped buildings whose end has passed.
"Difference array vs prefix sum?"Diff array supports range updates in O(1); prefix sum supports range queries in O(1). They are inverses.
