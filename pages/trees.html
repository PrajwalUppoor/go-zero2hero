Trees & BST

Traversals, DFS recursion that returns a useful local value, level-order BFS, BST properties, and serialize/deserialize.

Mental model

  • Trees are recursion's natural shape. Most tree problems collapse to: "what does the parent need to know about its subtree?".
  • The "DFS returns local value, closure stores global" trick handles diameter, max-path-sum, longest-univalue, balanced-check, house-robber-III.
  • Three traversals matter:
    • Pre-order: root before children (serialize, copy).
    • In-order: left, root, right (BST = sorted output).
    • Post-order: children before root (delete, aggregate).
  • Level-order = BFS with a queue. Process by levels using n := len(q) at the start of each iteration.
  • For BST, in-order traversal is sorted, so K-th smallest, validate, recover, range queries all map to "controlled in-order".

Node definition

type TreeNode struct {
    Val         int
    Left, Right *TreeNode

Traversals — iterative & recursive

// Inorder iterative (BST → sorted)
func inorder(root *TreeNode) []int {
    var res []int; var st []*TreeNode; cur := root
    for cur != nil || len(st) > 0 {
        for cur != nil { st = append(st, cur); cur = cur.Left }
        cur = st[len(st)-1]; st = st[:len(st)-1]
        res = append(res, cur.Val)
        cur = cur.Right
    }
    return res
}

// Level-order BFS
func levelOrder(root *TreeNode) [][]int {
    if root == nil { return nil }
    var res [][]int
    q := []*TreeNode{root}
    for len(q) > 0 {
        n := len(q); level := make([]int, 0, n)
        for i := 0; i < n; i++ {
            x := q[i]; level = append(level, x.Val)
            if x.Left != nil  { q = append(q, x.Left) }
            if x.Right != nil { q = append(q, x.Right) }
        }
        q = q[n:]
        res = append(res, level)
    }
    return res
}

The “return a useful local value” trick

// Diameter = longest path between any two nodes.
func diameter(root *TreeNode) int {
    best := 0
    var depth func(*TreeNode) int
    depth = func(n *TreeNode) int {
        if n == nil { return 0 }
        l, r := depth(n.Left), depth(n.Right)
        if l+r > best { best = l + r }
        return 1 + max(l, r)
    }
    depth(root)
    return best
}

// Max Path Sum — node may "open" a path or "close" one.
func maxPathSum(root *TreeNode) int {
    best := math.MinInt
    var gain func(*TreeNode) int
    gain = func(n *TreeNode) int {
        if n == nil { return 0 }
        l := max(gain(n.Left), 0)
        r := max(gain(n.Right), 0)
        if v := n.Val + l + r; v > best { best = v }
        return n.Val + max(l, r)
    }
    gain(root)
    return best
}
The recursive function returns what the parent needs (e.g. depth, best one-sided gain). The complete answer is captured in a closure-scoped variable.

BST core operations

// Validate BST with bounds (works for unique or non-unique semantics).
func isValidBST(root *TreeNode) bool {
    var dfs func(n *TreeNode, lo, hi int) bool
    dfs = func(n *TreeNode, lo, hi int) bool {
        if n == nil { return true }
        if n.Val <= lo || n.Val >= hi { return false }
        return dfs(n.Left, lo, n.Val) && dfs(n.Right, n.Val, hi)
    }
    return dfs(root, math.MinInt, math.MaxInt)
}

// K-th smallest via iterative inorder (stop early).
func kthSmallest(root *TreeNode, k int) int {
    var st []*TreeNode; cur := root
    for cur != nil || len(st) > 0 {
        for cur != nil { st = append(st, cur); cur = cur.Left }
        cur = st[len(st)-1]; st = st[:len(st)-1]
        k--; if k == 0 { return cur.Val }
        cur = cur.Right
    }
    return -1
}

Lowest Common Ancestor (binary tree)

func lca(root, p, q *TreeNode) *TreeNode {
    if root == nil || root == p || root == q { return root }
    l := lca(root.Left, p, q)
    r := lca(root.Right, p, q)
    if l != nil && r != nil { return root }
    if l != nil { return l }
    return r
}
  • In a BST, walk down: if both p, q < node go left; if both > node go right; else current is LCA. O(h).

Construct from traversals

// preorder + inorder uniquely determines a tree.
func build(pre, in []int) *TreeNode {
    idx := map[int]int{}
    for i, v := range in { idx[v] = i }
    pi := 0
    var rec func(l, r int) *TreeNode
    rec = func(l, r int) *TreeNode {
        if l > r { return nil }
        v := pre[pi]; pi++
        m := idx[v]
        return &TreeNode{Val: v, Left: rec(l, m-1), Right: rec(m+1, r)}
    }
    return rec(0, len(in)-1)
}

Serialize / deserialize

  • Pre-order with null markers — works for any binary tree.
  • For BST, you can skip nulls (bounds reconstruct structure).
  • Level-order is common in LeetCode I/O.

Anti-patterns — what NOT to say

  • "I'll modify global state across recursive calls." Closure variables only — globals don't reset between test cases.
  • Recursive depth-first on adversarial skewed trees — stack overflow at ~10⁵ depth. Switch to iterative + explicit stack.
  • Treating "binary tree" and "BST" interchangeably. BST gives you order; binary tree doesn't.
  • Using int bounds without min/max for Validate BST — pass math.MinInt/MaxInt sentinels.
  • Returning the wrong thing from the recursive helper — it should be the value the parent needs, not the answer itself.

Senior interviewer follow-ups

ProbeWhat they want to hear
"Iterative in-order traversal — how?"Stack: while node, push and go left; pop, visit, go right.
"Why does recursion work for LCA without storing parent pointers?"Each subtree returns a non-nil ancestor of any contained target; the first node where both subtrees return non-nil is the LCA.
"Serialize/deserialize: why pre-order with nulls?"Unique reconstruction with one pass; in-order alone doesn't work without the structural nulls.
"Validate BST without bounds?"In-order traversal must produce strictly increasing sequence.
"K-th smallest with frequent updates?"Augmented BST with subtree sizes (or fenwick over inserts).
"How to BFS without a queue (memory tight)?"Morris-like traversal with threading; iterative DFS with stack works for most.
