Arrays & Slices

The base of every other pattern. Learn the classics: prefix sums, two pointers, Kadane, Dutch flag, cyclic sort, in-place ops.

Mental model

  • Slices are the workhorse. Three fields under the hood — {ptr, len, cap} — and append doubles capacity when it grows.
  • Five canonical patterns cover ~80% of array problems: prefix sum, two pointers, sliding window, in-place write index, and Kadane.
  • If the array is sorted, lean on two pointers or binary search before anything else.
  • If the array contains values in [1..n], cyclic sort or index-as-marker tricks usually win.
  • If the constraint is "subarray sum equals K" (with negatives possible), reach for prefix-sum + hashmap of counts.

Core templates

1. Prefix sum

// Range sum [l, r] in O(1) after O(n) build.
func buildPrefix(a []int) []int {
    p := make([]int, len(a)+1)
    for i, v := range a { p[i+1] = p[i] + v }
    return p
}
func rangeSum(p []int, l, r int) int { return p[r+1] - p[l] }

2. Kadane (max subarray)

func maxSubArray(nums []int) int {
    best, cur := nums[0], nums[0]
    for i := 1; i < len(nums); i++ {
        if cur+nums[i] < nums[i] { cur = nums[i] } else { cur += nums[i] }
        if cur > best { best = cur }
    }
    return best
}

3. Dutch national flag (3-way partition)

func sortColors(a []int) {
    lo, mid, hi := 0, 0, len(a)-1
    for mid <= hi {
        switch a[mid] {
        case 0: a[lo], a[mid] = a[mid], a[lo]; lo++; mid++
        case 1: mid++
        case 2: a[mid], a[hi] = a[hi], a[mid]; hi--
        }
    }
}

4. Cyclic sort (numbers in [1..n])

func firstMissingPositive(a []int) int {
    n := len(a)
    for i := 0; i < n; {
        v := a[i]
        if v > 0 && v <= n && a[v-1] != v {
            a[i], a[v-1] = a[v-1], a[i]
        } else { i++ }
    }
    for i, v := range a { if v != i+1 { return i + 1 } }
    return n + 1
}

5. In-place reverse / rotate

func reverse(a []int, l, r int) {
    for l < r { a[l], a[r] = a[r], a[l]; l++; r-- }
}
// Rotate right by k:  reverse(0..n-1); reverse(0..k-1); reverse(k..n-1)
func rotate(a []int, k int) {
    n := len(a); k %= n
    reverse(a, 0, n-1); reverse(a, 0, k-1); reverse(a, k, n-1)
}

Anti-patterns — what NOT to say

  • "I'll just sort it." Sorting is O(n log n) — if a hash, two-pointer or prefix-sum solution exists at O(n), say so first.
  • "I'll allocate a new slice each iteration." Loop allocations destroy performance — pre-size with make([]T, 0, n).
  • Returning a sub-slice of a large internal buffer — caller can't garbage-collect the underlying array. slices.Clone at the boundary.
  • Manual loops where slices.Index, slices.Contains, slices.SortFunc are clearer.
  • Using a = append(a[:i], a[i+1:]...) in a hot loop — that's O(n) per delete; use a write index in O(n) total.

Senior interviewer follow-ups

ProbeWhat they want to hear
"Can you do Two Sum in O(1) extra space?"Only if input is sorted (then opposite-end two pointers).
"What if Maximum Subarray must return indices, not just sum?"Track curStart; reset when cur resets; also remember bestStart, bestEnd when you update best.
"How do you do Product Except Self with O(1) extra (output not counted)?"One pass of prefix products into output, second pass multiplies by running suffix.
"What if subarray sum can include negative numbers?"Sliding window fails; use prefix-sum + hashmap of counts.
"Rotate by k where k may be larger than n?"k %= n first; k may also be negative — normalize with ((k%n)+n)%n.
"Trapping Rain Water with O(1) space?"Two-pointer with running leftMax / rightMax; advance the smaller side.
"Streaming version of Top-K elements?"Min-heap of size K — replace top when next item is larger.

Senior pitfalls

  • Off-by-one on inclusive/exclusive ranges. Pick a convention (half-open [l, r)) and stick to it.
  • Mutating the caller's slice without documenting it.
  • Allocating in a hot loop — pre-size with make([]T, 0, n).
  • Slice aliasing bugs when returning subslices: copy if the caller may outlive the source.

Practice checklist

  • Implement prefix sum + range sum from memory.
  • Solve Subarray Sum = K with prefix-count map (negatives included).
  • Solve Trapping Rain Water two ways (stack + two-pointer).
  • Solve First Missing Positive in O(n) / O(1) extra.
  • Implement Kadane and adapt it to return indices, not just sum.
  • Sort Colors (Dutch flag) in one pass.
  • Rotate array right by k in O(n) / O(1).
  • Product Except Self without using division.
  • Find duplicates / missing using cyclic sort.
  • Boyer–Moore majority vote in one pass.
