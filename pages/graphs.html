Graphs — BFS, DFS, Topological Sort, Union-Find, Dijkstra

The most tactically rich category. Pick the right traversal first; then watch every other problem fall.

Mental model

  • Pick the right traversal first; the rest follows. Decision tree:
    • Unweighted shortest path → BFS.
    • Reachability / connectivity / cycles → DFS (or DSU for connectivity-only).
    • Ordering with prerequisites → topo sort (Kahn or DFS).
    • Weighted shortest path, non-negative → Dijkstra.
    • Weighted with negatives → Bellman-Ford / SPFA.
    • All-pairs (n ≤ 400) → Floyd-Warshall.
    • MST → Kruskal (sorted edges + DSU) or Prim.
  • Grids are graphs in disguise. 4- or 8-direction neighbors via dr/dc arrays. Flatten (r,c) to a single int via r*C+c if you need a hashable key.
  • For "components / merge / disjoint groups", Union-Find gives near-O(α(n)) per op — preferred over DFS when you only need connectivity.
  • Pre-compute in-degrees / adjacency before iterating, even if it costs an extra pass — the inner loop will thank you.

BFS — shortest path in an unweighted graph

func bfs(src int, adj [][]int) []int {
    n := len(adj)
    dist := make([]int, n); for i := range dist { dist[i] = -1 }
    dist[src] = 0
    q := []int{src}
    for len(q) > 0 {
        u := q[0]; q = q[1:]
        for _, v := range adj[u] {
            if dist[v] == -1 { dist[v] = dist[u] + 1; q = append(q, v) }
        }
    }
    return dist
}
  • Multi-source BFS: push every source into the queue at distance 0 (e.g. nearest 0, rotting oranges).
  • 0/1-BFS: deque, push-front for 0-weight, push-back for 1-weight.
  • Bi-directional BFS: from src and dst, meet in the middle — great for word ladder.

DFS template + 3-color cycle detection

const (white, gray, black = 0, 1, 2)

func hasCycleDirected(adj [][]int) bool {
    color := make([]int, len(adj))
    var dfs func(u int) bool
    dfs = func(u int) bool {
        color[u] = gray
        for _, v := range adj[u] {
            if color[v] == gray { return true }            // back-edge
            if color[v] == white && dfs(v) { return true }
        }
        color[u] = black
        return false
    }
    for u := range adj {
        if color[u] == white && dfs(u) { return true }
    }
    return false
}

Topological sort (Kahn / BFS)

func topo(n int, adj [][]int) ([]int, bool) {
    in := make([]int, n)
    for u := 0; u < n; u++ { for _, v := range adj[u] { in[v]++ } }
    var q []int
    for u := 0; u < n; u++ { if in[u] == 0 { q = append(q, u) } }
    order := make([]int, 0, n)
    for len(q) > 0 {
        u := q[0]; q = q[1:]; order = append(order, u)
        for _, v := range adj[u] {
            in[v]--
            if in[v] == 0 { q = append(q, v) }
        }
    }
    return order, len(order) == n  // false → cycle
}

Union-Find (DSU)

type DSU struct{ parent, size []int }
func NewDSU(n int) *DSU {
    p := make([]int, n); s := make([]int, n)
    for i := range p { p[i] = i; s[i] = 1 }
    return &DSU{p, s}
}
func (d *DSU) Find(x int) int {
    for d.parent[x] != x {
        d.parent[x] = d.parent[d.parent[x]]   // path compression (1-step)
        x = d.parent[x]
    }
    return x
}
func (d *DSU) Union(a, b int) bool {
    ra, rb := d.Find(a), d.Find(b)
    if ra == rb { return false }
    if d.size[ra] < d.size[rb] { ra, rb = rb, ra }
    d.parent[rb] = ra
    d.size[ra] += d.size[rb]
    return true
}
  • Use cases: connected components, redundant edge, accounts merge, Kruskal MST, friend circles.
  • Near O(α(n)) per op with path compression + union by size/rank.

Dijkstra (single-source, non-negative weights)

// Sketch — see Heaps page for full PQ definition.
func dijkstra(src, n int, adj [][]Edge) []int {
    const INF = int(1e18)
    dist := make([]int, n); for i := range dist { dist[i] = INF }
    dist[src] = 0
    pq := &PQ{{src, 0}}
    for pq.Len() > 0 {
        it := heap.Pop(pq).(PQItem)
        if it.Dist > dist[it.Node] { continue }
        for _, e := range adj[it.Node] {
            if nd := it.Dist + e.W; nd < dist[e.To] {
                dist[e.To] = nd
                heap.Push(pq, PQItem{e.To, nd})
            }
        }
    }
    return dist
}
  • Negative edges → Bellman-Ford (O(V·E)) or SPFA.
  • All-pairs (n ≤ 400) → Floyd-Warshall (O(n³)).
  • Single-source DAG → topological order + relax.

MST (Kruskal & Prim)

  • Kruskal: sort edges by weight, add if endpoints are in different DSU sets. O(E log E).
  • Prim: grow tree from a node using a heap of crossing edges. O(E log V).

Advanced topics worth knowing

  • Tarjan SCC / Kosaraju (two-pass on reverse graph).
  • Bridges & articulation points — DFS with disc/low arrays.
  • Bipartite check — BFS 2-coloring; failure ⇒ odd cycle.
  • Eulerian path — Hierholzer.
  • Maximum bipartite matching — Hopcroft–Karp (advanced).

Anti-patterns — what NOT to say

  • "I'll DFS to find the shortest path." DFS doesn't find shortest paths in unweighted graphs — BFS does.
  • Running Dijkstra on a graph with negative edges. Use Bellman-Ford.
  • Forgetting to mark before pushing in BFS — the same node gets enqueued multiple times.
  • Iterating map as a queue without preserving order.
  • Building an O(V²) adjacency matrix on a sparse graph (dump V=10⁵ in your face).
  • Recursing without cycle protection — infinite loop.

Senior interviewer follow-ups

ProbeWhat they want to hear
"BFS vs DFS for shortest path?"BFS only when all edges are equal weight (or 0/1 with deque-BFS); otherwise Dijkstra.
"How does Kahn's topo sort detect cycles?"If output length < V, some nodes never reached in-degree 0 → there's a cycle.
"Why path compression + union by size for DSU?"Combined gives near-constant α(n) per op; either alone is O(log n).
"Dijkstra correctness intuition?"Once a node is popped, no shorter path is possible; non-negative edges guarantee popped distance is final.
"Bidirectional BFS speedup?"Search from both ends; meeting in the middle reduces frontier from b^d to 2·b^(d/2).
"How to detect a cycle in an undirected graph?"DFS tracking parent (or DSU union returning false on common root).
"How does Bellman-Ford handle negative cycles?"Run V-1 relaxations; one more iteration that still relaxes ⇒ negative cycle reachable from source.
