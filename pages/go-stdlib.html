Stdlib & idiomatic Go

The packages every senior backend engineer must reach for instinctively: net/http, encoding/json, database/sql, testing, log/slog.

Mental model

  • The stdlib is the framework. A senior Go service typically pulls in < 10 third-party packages: a logger, a database driver, observability, maybe an HTTP router with patterns 1.22 doesn't yet cover.
  • Always pass context.Context as the first argument and honor it in every blocking call (QueryContext, NewRequestWithContext, l.Wait(ctx)).
  • Always set timeouts on http.Server, http.Client, database/sql. Defaults are infinite — production-hostile.
  • Reuse pooled clients (http.Client, *sql.DB). They are themselves the pool; one per dependency, package-level or struct field.
  • Structured logging (log/slog in 1.21+) is non-negotiable. Strings are for humans; JSON is for machines.

net/http server you can read in your sleep

type Server struct {
    repo Repo
    log  *slog.Logger
}

func (s *Server) routes() http.Handler {
    mux := http.NewServeMux()
    mux.HandleFunc("GET /users/{id}", s.getUser)
    mux.HandleFunc("POST /users",     s.createUser)
    return s.withRecover(s.withLogging(mux))
}

func (s *Server) getUser(w http.ResponseWriter, r *http.Request) {
    id := r.PathValue("id")
    u, err := s.repo.Get(r.Context(), id)
    if errors.Is(err, ErrNotFound) { http.Error(w, "not found", 404); return }
    if err != nil                  { http.Error(w, "boom", 500); return }
    w.Header().Set("Content-Type", "application/json")
    _ = json.NewEncoder(w).Encode(u)
}

func main() {
    srv := &http.Server{
        Addr:              ":8080",
        Handler:           (&Server{}).routes(),
        ReadHeaderTimeout: 5 * time.Second,
        ReadTimeout:       10 * time.Second,
        WriteTimeout:      10 * time.Second,
        IdleTimeout:       60 * time.Second,
    }
    log.Fatal(srv.ListenAndServe())
}
Always set timeouts on http.Server. Defaults are unlimited and a slowloris will eat all your goroutines.

Graceful shutdown

ctx, stop := signal.NotifyContext(context.Background(), os.Interrupt, syscall.SIGTERM)
defer stop()

go func() { _ = srv.ListenAndServe() }()
<-ctx.Done()

shutdownCtx, cancel := context.WithTimeout(context.Background(), 15*time.Second)
defer cancel()
_ = srv.Shutdown(shutdownCtx)

Middleware composition

func (s *Server) withLogging(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        start := time.Now()
        next.ServeHTTP(w, r)
        s.log.Info("req", "path", r.URL.Path, "ms", time.Since(start).Milliseconds())
    })
}

func (s *Server) withRecover(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        defer func() {
            if rec := recover(); rec != nil {
                s.log.Error("panic", "err", rec, "stack", string(debug.Stack()))
                http.Error(w, "internal", 500)
            }
        }()
        next.ServeHTTP(w, r)
    })
}

HTTP client done right

var httpClient = &http.Client{
    Timeout: 5 * time.Second,
    Transport: &http.Transport{
        MaxIdleConns:        100,
        MaxIdleConnsPerHost: 10,
        IdleConnTimeout:     90 * time.Second,
    },
}
  • Reuse a single client: each http.Client has its own pool.
  • Always read & close the response body to return the connection to the pool.

encoding/json

type User struct {
    ID    uint64 `json:"id"`
    Name  string `json:"name"`
    Email string `json:"email,omitempty"`
    pwd   string `json:"-"`           // unexported, never serialized
}

// Stream large payloads
dec := json.NewDecoder(r.Body)
dec.DisallowUnknownFields()
var u User
if err := dec.Decode(&u); err != nil { ... }
For very large payloads or hot paths, consider jsoniter, go-json, or codegen (easyjson, ffjson). Profile first.

database/sql

db, err := sql.Open("postgres", dsn)
if err != nil { return err }
db.SetMaxOpenConns(25)
db.SetMaxIdleConns(25)
db.SetConnMaxLifetime(5 * time.Minute)

ctx, cancel := context.WithTimeout(ctx, 500*time.Millisecond)
defer cancel()

row := db.QueryRowContext(ctx, "SELECT name FROM users WHERE id=$1", id)
var name string
if err := row.Scan(&name); err != nil {
    if errors.Is(err, sql.ErrNoRows) { return "", ErrNotFound }
    return "", err
}
  • Always use the Context variants — they cancel queries on client disconnect.
  • Never fmt.Sprintf values into SQL. Always parameterize.
  • Use sqlx, pgx, sqlc for ergonomics; database/sql alone is fine for simple cases.

testing — the patterns reviewers expect

func TestParse(t *testing.T) {
    cases := []struct{
        name string
        in   string
        want int
        err  bool
    }{
        {"empty", "",  0, true},
        {"one",   "1", 1, false},
        {"big",   "9999999999", 0, true},
    }
    for _, tc := range cases {
        tc := tc
        t.Run(tc.name, func(t *testing.T) {
            t.Parallel()
            got, err := parse(tc.in)
            if (err != nil) != tc.err { t.Fatalf("err=%v wantErr=%v", err, tc.err) }
            if got != tc.want { t.Errorf("got %d want %d", got, tc.want) }
        })
    }
}
  • Table-driven tests with subtests; t.Parallel() where independent.
  • testing.T.Helper() on assertion helpers for clean failure lines.
  • t.TempDir(), t.Setenv() for hermetic tests.
  • Fuzz: func FuzzParse(f *testing.F) for parsers.

Anti-patterns — what NOT to say

  • "I'll use the default http.Client." Default has no timeout — one slow upstream and your service hangs forever.
  • Building SQL with fmt.Sprintf. Always parameterize.
  • Logging with fmt.Println or log.Printf in services. Use slog (or zap) so you can grep/filter in production.
  • "defer rows.Close() at the start, then forget the if err := rows.Err()." Always check terminal error after the loop.
  • Reading the response body and forgetting to close it — connection never returns to pool, eventually exhausts.
  • Catching recover() and ignoring it. At minimum log + stack-trace, return 500.
  • Encoding/decoding huge payloads with json.Marshal instead of streaming via Encoder/Decoder.

Senior interviewer follow-ups

ProbeWhat they want to hear
"How do you tune database/sql?"Set MaxOpenConns ≤ DB-side limit/instances; MaxIdleConns ≈ MaxOpenConns; ConnMaxLifetime < the load-balancer/server idle close.
"How do you propagate cancellation through HTTP middleware?"Always use r.Context() derived contexts; pass the same ctx to downstream http.NewRequestWithContext and db.QueryContext.
"Difference between http.HandleFunc and a custom mux?"Default mux is global state — bad for tests. Always use a fresh http.NewServeMux() in routes().
"How do you stream a large JSON list?"json.NewEncoder(w).Encode per item, framed in a JSON-lines body, or chunked array with manual delimiters.
"What does DisallowUnknownFields() protect against?"Silently ignored typos in client payloads — useful for strict contracts.
"How do you test an HTTP handler?"httptest.NewServer + real http.Client, or httptest.NewRecorder + direct ServeHTTP. Subtest per case, t.Parallel().
"What does http.Server.Shutdown do?"Stops accepting new connections, waits for in-flight handlers to finish (or ctx deadline). It does not kill long-lived hijacked conns — close those yourself.

Self-check

  • I can write an HTTP server with timeouts, mux, middleware, and graceful shutdown from memory.
  • I always use the Context variants of database/sql, http.NewRequest, and errgroup.
  • I know how to tune db.SetMaxOpenConns, SetMaxIdleConns, SetConnMaxLifetime.
  • I write table-driven subtests with t.Parallel() and t.Helper().
  • I can stream JSON in/out using Decoder/Encoder.
  • I emit structured logs with slog and a request-scoped logger via context.
  • I close response bodies and check rows.Err() after every loop.
  • I never fmt.Sprintf values into SQL.
  • I have a withRecover middleware that logs and returns 500.
  • I know how to write a fuzz test for a parser.
