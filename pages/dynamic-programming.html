Dynamic Programming

The pattern that gates senior offers. Internalize the families — 1D, 2D, knapsack, LIS, interval, tree, bitmask, digit — and the rest is parameter shuffling.

Mental model

  • The DP signature: optimal substructure + overlapping subproblems. Optimal substructure alone = greedy / divide-conquer. Overlap is what forces memoization.
  • Identify the family in 30 seconds: linear sequence (Fibonacci/rob), knapsack (subset constraint), LIS (longest-increasing), 2D string DP (LCS/edit), interval (split + combine), tree (DFS returning state), bitmask (n ≤ 20), digit, stocks (k transactions × holding state).
  • Top-down memo first, bottom-up only if you need to optimize. Top-down is more natural to derive; bottom-up unlocks 1D rolling and is sometimes faster.
  • Always count states × work per state before coding. If |S| · w(S) > 10^8, you need a tighter state.
  • Roll the array: when dp[i] only depends on dp[i-1] (or a small window), drop the dimension. prev2, prev1 patterns.

The 5-step framework

  1. Define state: the smallest tuple that fully describes a sub-problem.
  2. Recurrence: how does state combine smaller states?
  3. Base case(s): the smallest non-trivial state.
  4. Order of computation: top-down memo or bottom-up table.
  5. Optimize: roll the table to 1D if possible; compress state.

Family 1 — 1D linear DP

Fibonacci / Climbing Stairs / House Robber

func rob(a []int) int {
    prev2, prev1 := 0, 0
    for _, v := range a {
        prev2, prev1 = prev1, max(prev1, prev2 + v)
    }
    return prev1
}

Coin Change (min coins, unbounded)

func coinChange(coins []int, amount int) int {
    const INF = math.MaxInt / 2
    dp := make([]int, amount+1)
    for i := range dp { dp[i] = INF }; dp[0] = 0
    for v := 1; v <= amount; v++ {
        for _, c := range coins {
            if c <= v && dp[v-c]+1 < dp[v] { dp[v] = dp[v-c] + 1 }
        }
    }
    if dp[amount] == INF { return -1 }
    return dp[amount]
}

Word Break

func wordBreak(s string, dict []string) bool {
    set := map[string]bool{}; for _, w := range dict { set[w] = true }
    dp := make([]bool, len(s)+1); dp[0] = true
    for i := 1; i <= len(s); i++ {
        for j := 0; j < i; j++ {
            if dp[j] && set[s[j:i]] { dp[i] = true; break }
        }
    }
    return dp[len(s)]
}

Family 2 — Knapsack

0/1 knapsack (1D rolling)

func knapsack01(w, v []int, W int) int {
    dp := make([]int, W+1)
    for i := range w {
        for c := W; c >= w[i]; c-- {           // reverse capacity
            if dp[c-w[i]]+v[i] > dp[c] { dp[c] = dp[c-w[i]] + v[i] }
        }
    }
    return dp[W]
}

Unbounded knapsack

// Same shape, capacity loop ASCENDING — allows reuse of the same item.
for c := w[i]; c <= W; c++ {
    if dp[c-w[i]]+v[i] > dp[c] { dp[c] = dp[c-w[i]] + v[i] }
}

Subset Sum / Partition Equal Subset Sum / Target Sum

  • Reduce to "can we hit a target?" → 1D boolean DP.
  • Target Sum: convert to subset-sum on (total+target)/2.

Family 3 — Longest Increasing Subsequence

// O(n log n): tails[i] = smallest tail of any increasing subseq of length i+1
func lengthOfLIS(a []int) int {
    var tails []int
    for _, x := range a {
        i := sort.SearchInts(tails, x)         // strict increase; use SearchInts(tails, x+1) for <=
        if i == len(tails) { tails = append(tails, x) } else { tails[i] = x }
    }
    return len(tails)
}
  • Russian Doll Envelopes → sort by width asc, height desc; LIS on heights.
  • Longest Bitonic Subsequence → LIS left-to-right + LIS right-to-left.

Family 4 — 2D DP on strings

Longest Common Subsequence

func lcs(a, b string) int {
    m, n := len(a), len(b)
    dp := make([][]int, m+1); for i := range dp { dp[i] = make([]int, n+1) }
    for i := 1; i <= m; i++ {
        for j := 1; j <= n; j++ {
            if a[i-1] == b[j-1] { dp[i][j] = dp[i-1][j-1] + 1 } else {
                if dp[i-1][j] > dp[i][j-1] { dp[i][j] = dp[i-1][j] } else { dp[i][j] = dp[i][j-1] }
            }
        }
    }
    return dp[m][n]
}

Edit Distance

// dp[i][j] = min edits to convert a[:i] to b[:j]
// match → dp[i-1][j-1];  else 1 + min(dp[i-1][j-1] sub, dp[i-1][j] del, dp[i][j-1] ins)

Regular Expression / Wildcard Matching

  • Track (i, j) over (string, pattern). Cases for '.', '*', '?'.
  • '*' (LeetCode regex): either match zero (dp[i][j-2]) or one more (dp[i-1][j] if p[j-2] matches s[i-1]).

Family 5 — Interval DP

  • State dp[l][r], transition splits on k in (l, r).
  • Outer loop = length, inner = left, innermost = k.
  • Examples: Matrix Chain Multiplication, Burst Balloons, Stone Game, Minimum Cost to Cut a Stick.
// Burst Balloons — pad with 1s; dp[l][r] = max coins by bursting last in (l, r).
for length := 2; length < len(nums); length++ {
    for l := 0; l+length < len(nums); l++ {
        r := l + length
        best := 0
        for k := l + 1; k < r; k++ {
            v := nums[l]*nums[k]*nums[r] + dp[l][k] + dp[k][r]
            if v > best { best = v }
        }
        dp[l][r] = best
    }
}

Family 6 — Tree DP

// House Robber III — return (robThis, skipThis) up the tree.
func rob(root *TreeNode) int {
    var dfs func(n *TreeNode) (int, int)
    dfs = func(n *TreeNode) (int, int) {
        if n == nil { return 0, 0 }
        lr, ls := dfs(n.Left)
        rr, rs := dfs(n.Right)
        return n.Val + ls + rs, max(lr, ls) + max(rr, rs)
    }
    a, b := dfs(root); return max(a, b)
}

Family 7 — Bitmask DP (n ≤ 20)

  • State is a subset of n elements; transitions add one element.
  • Example: Travelling Salesman Person → dp[mask][i] = min over j (dp[mask without i][j] + d[j][i]).
// Enumerate submasks of mask in O(3^n) total:
for sub := mask; sub > 0; sub = (sub - 1) & mask { /* ... */ }

Family 8 — Digit DP

  • Count numbers ≤ N with some digit property.
  • State: (pos, tight, leadingZero, customState). Memo on (pos, customState) when not tight.

Family 9 — DP on stocks

  • Generalize to dp[i][k][holding]: i = day, k = remaining transactions, holding ∈ {0,1}.
  • Recurrence: dp[i][k][0] = max(dp[i-1][k][0], dp[i-1][k][1] + p[i]); symmetric for holding=1.
  • Cooldown → use a 3-state machine (hold, sold, rest).

Top-down (memo) template

memo := map[State]int{}
var rec func(s State) int
rec = func(s State) int {
    if base(s) { return baseVal }
    if v, ok := memo[s]; ok { return v }
    best := worst
    for _, ns := range nextStates(s) {
        if v := transition(s, ns) + rec(ns); better(v, best) { best = v }
    }
    memo[s] = best
    return best
}

Anti-patterns — what NOT to say

  • "DP is just memoization." DP requires identifying the right state first; memoization is a mechanism.
  • Using map as memo when an array would do. Maps cost ~10x more for tight inner loops.
  • Iterating capacity ascending in 0/1 knapsack — that's unbounded; you'll allow item reuse.
  • State that includes everything "just in case." Each extra dimension multiplies state count.
  • Recomputing transitions inside the inner loop when they could be hoisted.
  • Building a 2D dp when only the previous row is needed — wastes O(mn) when O(n) works.

Senior interviewer follow-ups

ProbeWhat they want to hear
"What state did you pick and why?"The smallest tuple s.t. dp(s) is independent of the path that reached it. Justify the dimensions.
"Top-down vs bottom-up?"Top-down = recursion + memo, easy to derive. Bottom-up unlocks rolling-array space optimization and avoids stack depth.
"Why does 0/1 knapsack iterate capacity in reverse?"To prevent reusing the same item (we depend on the previous iteration's value).
"How is LIS in O(n log n)?"Patience sorting — maintain tails[k] = smallest tail of any increasing subseq of length k+1; binary-search insert.
"How do you reconstruct the DP solution, not just the value?"Either store choice in a parallel choice[i][j] table, or backtrack through the dp table.
"How do you parallelize / cache-tile a 2D DP?"Process anti-diagonals or tiles to keep the working set in cache.
"When is bitmask DP worth it?"n ≤ 20 and the problem asks about subsets/orderings — TSP, assignment, watchworks.

Pitfalls

  • Wrong state → exponential blowup. Always count states × work per state before coding.
  • Loop direction in 0/1 knapsack — reversing matters.
  • Off-by-one between “index up to i” and “first i items”.
  • Maps as memo can be slower than arrays for dense state spaces.
