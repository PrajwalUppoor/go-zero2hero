Stacks & Queues

Use a slice for both. The interesting stuff is monotonic stacks and monotonic deques — they collapse O(n²) into O(n).

Mental model

  • Stack = "depend on the most recent thing": parens, recursion-on-iterative, expression eval, undo. LIFO.
  • Queue = "process in order of arrival": BFS, schedulers, level-order. FIFO.
  • Monotonic stack/deque = "next greater / smaller / max-in-window". They are the unlock for many "next-X-element" problems, collapsing O(n²) brute force to O(n).
  • Two-stack queue / two-queue stack: classic interview puzzle that's also a real technique for amortized O(1) ops with simple data structures.
  • Decode-string, basic-calculator, asteroid-collision: all stack problems wearing different costumes.

Idiomatic Go containers

// Stack
var st []int
st = append(st, x)                   // push
top := st[len(st)-1]                 // peek
st = st[:len(st)-1]                  // pop

// Queue (FIFO)
var q []int
q = append(q, x)                     // enqueue
head := q[0]; q = q[1:]              // dequeue  (O(1) amortized for small n)

// For large queues prefer container/list or a ring buffer to avoid retaining memory.

Valid parentheses

func isValid(s string) bool {
    pair := map[byte]byte{')': '(', ']': '[', '}': '{'}
    var st []byte
    for i := 0; i < len(s); i++ {
        c := s[i]
        if c == '(' || c == '[' || c == '{' { st = append(st, c); continue }
        if len(st) == 0 || st[len(st)-1] != pair[c] { return false }
        st = st[:len(st)-1]
    }
    return len(st) == 0
}

Monotonic stack — Next Greater Element

func nextGreater(a []int) []int {
    res := make([]int, len(a))
    for i := range res { res[i] = -1 }
    var st []int                     // indices, values decreasing
    for i, v := range a {
        for len(st) > 0 && a[st[len(st)-1]] < v {
            res[st[len(st)-1]] = v
            st = st[:len(st)-1]
        }
        st = append(st, i)
    }
    return res
}
Mnemonic: decreasing stack → next greater. Increasing stack → next smaller. Push indices; pop while the new element breaks monotonicity.

Largest Rectangle in Histogram

func largestRectangleArea(h []int) int {
    h = append(h, 0)                 // sentinel flushes the stack
    var st []int; best := 0
    for i, v := range h {
        for len(st) > 0 && h[st[len(st)-1]] > v {
            top := st[len(st)-1]; st = st[:len(st)-1]
            width := i
            if len(st) > 0 { width = i - st[len(st)-1] - 1 }
            if a := h[top] * width; a > best { best = a }
        }
        st = append(st, i)
    }
    return best
}

Sliding-window maximum (monotonic deque)

func maxSlidingWindow(a []int, k int) []int {
    var dq []int
    res := make([]int, 0, len(a)-k+1)
    for i, v := range a {
        for len(dq) > 0 && a[dq[len(dq)-1]] < v { dq = dq[:len(dq)-1] }
        dq = append(dq, i)
        if dq[0] <= i-k { dq = dq[1:] }
        if i >= k-1 { res = append(res, a[dq[0]]) }
    }
    return res
}

Queue using two stacks

type MyQueue struct{ in, out []int }
func (q *MyQueue) Push(x int) { q.in = append(q.in, x) }
func (q *MyQueue) Pop() int {
    q.shift()
    v := q.out[len(q.out)-1]; q.out = q.out[:len(q.out)-1]; return v
}
func (q *MyQueue) Peek() int { q.shift(); return q.out[len(q.out)-1] }
func (q *MyQueue) Empty() bool { return len(q.in) == 0 && len(q.out) == 0 }
func (q *MyQueue) shift() {
    if len(q.out) == 0 {
        for i := len(q.in) - 1; i >= 0; i-- { q.out = append(q.out, q.in[i]) }
        q.in = q.in[:0]
    }
}

Min stack (constant extra per push)

type MinStack struct{ data []int; mins []int }
func (s *MinStack) Push(x int) {
    s.data = append(s.data, x)
    m := x
    if len(s.mins) > 0 && s.mins[len(s.mins)-1] < m { m = s.mins[len(s.mins)-1] }
    s.mins = append(s.mins, m)
}
func (s *MinStack) Pop()    { s.data = s.data[:len(s.data)-1]; s.mins = s.mins[:len(s.mins)-1] }
func (s *MinStack) Top() int { return s.data[len(s.data)-1] }
func (s *MinStack) Min() int { return s.mins[len(s.mins)-1] }

Anti-patterns — what NOT to say

  • "I'll use container/list for a stack." A slice is faster, simpler, and idiomatic.
  • Re-scanning the array to find next greater. That's O(n²) — use a monotonic stack.
  • Building an "array queue" by repeatedly slicing q = q[1:] on huge data. Eventually the underlying array isn't reclaimed; use a head index or a ring buffer.
  • Treating "balanced parens" with counters when there are multiple bracket types — only stack handles nesting correctly.
  • Forgetting the sentinel zero at the end of the histogram for Largest Rectangle.

Senior interviewer follow-ups

ProbeWhat they want to hear
"Why is monotonic stack amortized O(n)?"Each index pushed once, popped at most once.
"Decreasing or increasing — which gives next greater?"Decreasing stack: when a bigger element arrives, popped items get their next-greater answered.
"Implement a queue using two stacks — pop cost?"Amortized O(1); worst case O(n) when out-stack drains in.
"How do you handle Basic Calculator with parens and *,/?"Two stacks (nums, ops), or recursive descent: paren = recurse, then evaluate left-to-right respecting precedence.
"Streaming median?"Two heaps (max-heap of lower half, min-heap of upper half) — not stack, but worth mentioning.
"Sliding-window minimum with negatives?"Same monotonic deque, increasing instead of decreasing.
