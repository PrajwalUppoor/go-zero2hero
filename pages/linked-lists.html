Linked Lists

Every linked-list problem is a 3-pointer dance (prev, cur, next) plus a dummy head. Master fast/slow, reverse, and merge — the rest is recombination.

Mental model

  • Three pointers + a dummy head solve almost everything. prev for stitching, cur for "where I am", next saved before mutating cur.Next.
  • Five primitives compose all linked-list problems: reverse, find middle, detect cycle (Floyd), merge two sorted, k-th from end (gap pointer).
  • If a problem requires "two passes minimum" (like palindrome, reorder), it's "find middle, reverse second half, do something".
  • If a problem is about "K" (Kth from end, rotate by K, reverse in groups of K), it's the gap-pointer trick.
  • Watch for memory ownership. In Go, dangling prev.Next assignments can keep entire sublists alive. Set to nil when isolating.

Node definition

type ListNode struct {
    Val  int
    Next *ListNode
}

Universal trick: dummy head

dummy := &ListNode{Next: head}
prev := dummy
// ... operate via prev ...
return dummy.Next

Reverse a list (iterative)

func reverse(head *ListNode) *ListNode {
    var prev *ListNode
    cur := head
    for cur != nil {
        nxt := cur.Next
        cur.Next = prev
        prev = cur
        cur = nxt
    }
    return prev
}

Reverse a sublist [l, r]

func reverseBetween(head *ListNode, left, right int) *ListNode {
    dummy := &ListNode{Next: head}
    prev := dummy
    for i := 1; i < left; i++ { prev = prev.Next }
    cur := prev.Next
    for i := 0; i < right-left; i++ {
        nxt := cur.Next
        cur.Next = nxt.Next
        nxt.Next = prev.Next
        prev.Next = nxt
    }
    return dummy.Next
}

Cycle detection (Floyd)

func detectCycle(head *ListNode) *ListNode {
    slow, fast := head, head
    for fast != nil && fast.Next != nil {
        slow, fast = slow.Next, fast.Next.Next
        if slow == fast {
            p := head
            for p != slow { p = p.Next; slow = slow.Next }
            return p
        }
    }
    return nil
}

Find middle

// Slow ends on the (n/2 + 1)-th node for even length (right of center).
slow, fast := head, head
for fast != nil && fast.Next != nil { slow, fast = slow.Next, fast.Next.Next }

Merge two sorted lists

func merge(a, b *ListNode) *ListNode {
    dummy := &ListNode{}; tail := dummy
    for a != nil && b != nil {
        if a.Val < b.Val { tail.Next = a; a = a.Next } else { tail.Next = b; b = b.Next }
        tail = tail.Next
    }
    if a != nil { tail.Next = a } else { tail.Next = b }
    return dummy.Next
}

Merge K sorted lists (heap)

// Heap of (val, listIdx); pop, append to tail, push next of that list.
// O(N log k) where N = total nodes, k = #lists.

Add two numbers

func addTwoNumbers(a, b *ListNode) *ListNode {
    dummy := &ListNode{}; tail := dummy; carry := 0
    for a != nil || b != nil || carry > 0 {
        s := carry
        if a != nil { s += a.Val; a = a.Next }
        if b != nil { s += b.Val; b = b.Next }
        carry = s / 10
        tail.Next = &ListNode{Val: s % 10}
        tail = tail.Next
    }
    return dummy.Next
}

Reorder / partition / rotate

  • Reorder L0→Ln→L1→… — find middle, reverse second half, weave.
  • Palindrome check — same trick: middle + reverse + compare.
  • Partition around x — two dummy heads (less, geq), concatenate.
  • Rotate right by k — close the loop, walk to new tail, cut.

Anti-patterns — what NOT to say

  • Recursing on a million-node list — stack overflow. Iterate.
  • Mutating cur.Next before saving next := cur.Next — guaranteed lost-pointer bug.
  • Forgetting the dummy head and writing 5 special cases for "is the first node included?".
  • Using a slice to "make linked-list problems easier" — the interviewer wants pointer manipulation.
  • Returning head from a function that may have made a new head — return dummy.Next instead.

Senior interviewer follow-ups

ProbeWhat they want to hear
"Why does Floyd's tortoise-hare find the cycle entry?"Algebra: distance from head to entry equals distance from meeting point to entry (mod cycle length).
"How do you reverse in groups of K with O(1) extra space?"Iterate, count K, reverse that segment in place, link to previous segment's tail.
"What's O(1) Copy with Random Pointer?"Interleave each clone after its original; fix random pointers via orig.next.random = orig.random.next; then unweave.
"How does LRU achieve O(1)?"Hashmap of key → node ptr in a doubly-linked list. Touch = move-to-front; evict = pop tail.
"Detect intersection without modifying the lists?"Two pointers; when one hits null, switch to the other head. Both travel m+n; meet at intersection.
"Sort linked list in O(n log n) / O(1) extra?"Bottom-up merge sort: width 1, 2, 4, 8, ... merge pairs in place.

Self-check

  • I can reverse a list iteratively, no peeking.
  • I can find the middle (with the right tie-break for even-length).
  • I can detect cycle and find its entry node.
  • I can merge two sorted lists with a dummy.
  • I can remove N-th from end in one pass.
  • I can reorder / check palindrome with mid + reverse + weave/compare.
  • I can implement an LRU cache from scratch in O(1).
  • I can copy a list with random pointers in O(1) extra space.
  • I can reverse in groups of K, iterative.
  • I can find intersection of two lists without a hashset.
