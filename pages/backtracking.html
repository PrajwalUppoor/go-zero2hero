Backtracking

“Generate all / find any / enumerate every…”. Choose → recurse → un-choose. Prune ruthlessly, copy when saving.

Mental model

  • The trigger: "all subsets / permutations / combinations / partitions / arrangements / valid sequences / generate".
  • The structure: choose → recurse → un-choose. Always copy the path before saving (the slice is mutated in place).
  • Three flavors:
    1. Subsets / combinations — pick or skip each item; pass an i to avoid revisiting.
    2. Permutations — track used[] so each element appears once per arrangement.
    3. Constraint problems (N-Queens, Sudoku, Word Search) — propagate constraints; backtrack only on failure.
  • Pruning is the difference between TLE and accepted: sort first, skip duplicates, break on remaining-too-small.
  • If the recursion has overlapping subproblems, it's DP — memoize and lift.

Universal template

func backtrack(path []T, choices ...) {
    if isGoal(path) {
        save(append([]T(nil), path...))   // copy before storing
        return
    }
    for _, c := range nextChoices(path) {
        if !valid(path, c) { continue }
        path = append(path, c)
        backtrack(path, ...)
        path = path[:len(path)-1]         // un-choose
    }
}

Subsets (combinations of all sizes)

func subsets(nums []int) [][]int {
    var res [][]int
    var path []int
    var rec func(i int)
    rec = func(i int) {
        res = append(res, append([]int(nil), path...))
        for j := i; j < len(nums); j++ {
            path = append(path, nums[j])
            rec(j + 1)
            path = path[:len(path)-1]
        }
    }
    rec(0)
    return res
}

Permutations

func permute(nums []int) [][]int {
    var res [][]int
    used := make([]bool, len(nums))
    var path []int
    var rec func()
    rec = func() {
        if len(path) == len(nums) {
            res = append(res, append([]int(nil), path...))
            return
        }
        for i, v := range nums {
            if used[i] { continue }
            used[i] = true; path = append(path, v)
            rec()
            used[i] = false; path = path[:len(path)-1]
        }
    }
    rec()
    return res
}

Combination Sum (unbounded reuse)

func combinationSum(nums []int, target int) [][]int {
    sort.Ints(nums)
    var res [][]int; var path []int
    var rec func(i, rem int)
    rec = func(i, rem int) {
        if rem == 0 { res = append(res, append([]int(nil), path...)); return }
        for j := i; j < len(nums) && nums[j] <= rem; j++ {
            path = append(path, nums[j])
            rec(j, rem-nums[j])          // j (reuse), not j+1
            path = path[:len(path)-1]
        }
    }
    rec(0, target)
    return res
}

N-Queens

func solveNQueens(n int) [][]string {
    cols := make([]bool, n)
    d1   := make([]bool, 2*n)           // r + c
    d2   := make([]bool, 2*n)           // r - c + n
    pos  := make([]int, n)
    var res [][]string
    var rec func(r int)
    rec = func(r int) {
        if r == n {
            board := make([]string, n)
            for i, c := range pos {
                row := make([]byte, n); for k := range row { row[k] = '.' }
                row[c] = 'Q'; board[i] = string(row)
            }
            res = append(res, board); return
        }
        for c := 0; c < n; c++ {
            if cols[c] || d1[r+c] || d2[r-c+n] { continue }
            cols[c], d1[r+c], d2[r-c+n] = true, true, true
            pos[r] = c
            rec(r + 1)
            cols[c], d1[r+c], d2[r-c+n] = false, false, false
        }
    }
    rec(0)
    return res
}

Word Search (DFS on grid)

func exist(board [][]byte, word string) bool {
    R, C := len(board), len(board[0])
    var dfs func(r, c, i int) bool
    dfs = func(r, c, i int) bool {
        if i == len(word) { return true }
        if r < 0 || r >= R || c < 0 || c >= C || board[r][c] != word[i] { return false }
        board[r][c] = '#'                 // mark visited
        ok := dfs(r+1,c,i+1) || dfs(r-1,c,i+1) || dfs(r,c+1,i+1) || dfs(r,c-1,i+1)
        board[r][c] = word[i]             // restore
        return ok
    }
    for r := 0; r < R; r++ {
        for c := 0; c < C; c++ { if dfs(r, c, 0) { return true } }
    }
    return false
}

Pruning tactics

  • Sort first, then break as soon as remaining choice ≥ remaining budget.
  • Skip duplicates: after sorting, in combination/subset problems use if j > start && nums[j] == nums[j-1] { continue }.
  • Constraint propagation: Sudoku — recurse on the cell with fewest valid options.
  • Memoize sub-states when there's overlap (becomes top-down DP).

Anti-patterns — what NOT to say

  • res = append(res, path) without copying — every entry will be aliased to the same slice.
  • Forgetting to un-choose after recursion — leaks state into siblings.
  • Skipping the sort step on inputs with duplicates — duplicate sub-results explode.
  • Recursing without depth bound on adversarial inputs — stack overflow at ~10⁵.
  • Using backtracking when DP would be fine — exponential when O(n²) is on the table.

Senior interviewer follow-ups

ProbeWhat they want to hear
"How do you handle duplicates in Subsets II?"Sort, then skip nums[j] == nums[j-1] within the same recursion level (not across).
"Permutations II without sorting?"Sort + skip used[i-1] == false && nums[i] == nums[i-1].
"Estimated time complexity of backtracking?"Branching factor × depth — typically O(b^d). State + dedup may reduce significantly.
"When is backtracking actually DP?"When subproblems repeat — same state reached via different paths. Memoize the state.
"Why mark-and-restore in Word Search?"O(1) extra space — beats a separate visited matrix per call.
"Sudoku — most-constrained-variable first?"Pick the empty cell with fewest valid digits. Drastically reduces branching.
