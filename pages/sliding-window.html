Sliding Window

Two flavors: fixed-size (k known) and variable-size (grow until invalid, then shrink). Both are O(n) when each element enters and leaves the window once.

Mental model

  • The hint: "longest/shortest contiguous subarray/substring satisfying X" → sliding window.
  • The invariant: each element enters the window once and leaves once. That makes it amortized O(n) even with a nested while-loop inside.
  • Two flavors:
    • Fixed-size k — slide a fixed window, recompute aggregate by add-new / drop-old.
    • Variable-size — grow until invalid (or until you have a candidate), shrink to restore the invariant. Track best window length / sum / count.
  • The "exactly K" trick: when "exactly K distinct" is hard, compute "≤ K" – "≤ K-1".
  • If the constraint is non-monotonic (e.g., negatives in a sum problem) → switch to prefix-sum + map.

Variable-size template

// Find smallest/longest window satisfying a condition on counts/sum.
func template(nums []int) int {
    l, best := 0, 0
    state := newState()
    for r := 0; r < len(nums); r++ {
        state.add(nums[r])
        for !state.valid() {                // shrink until valid
            state.remove(nums[l])
            l++
        }
        if r-l+1 > best { best = r-l+1 }    // or update min
    }
    return best
}

Fixed-size template

// Window of size k. Add nums[r], drop nums[r-k] when r >= k.
func maxAvgSubarray(a []int, k int) float64 {
    sum, best := 0, math.MinInt64
    for i := 0; i < len(a); i++ {
        sum += a[i]
        if i >= k { sum -= a[i-k] }
        if i >= k-1 && sum > best { best = sum }
    }
    return float64(best) / float64(k)
}

Worked examples

Longest substring without repeating characters

func lengthOfLongestSubstring(s string) int {
    last := make(map[byte]int)
    l, best := 0, 0
    for r := 0; r < len(s); r++ {
        if i, ok := last[s[r]]; ok && i >= l { l = i + 1 }
        last[s[r]] = r
        if r-l+1 > best { best = r-l+1 }
    }
    return best
}

Minimum window substring

func minWindow(s, t string) string {
    if len(s) < len(t) { return "" }
    var need [128]int
    for i := 0; i < len(t); i++ { need[t[i]]++ }
    missing := len(t)
    bl, br := -1, 0
    l := 0
    for r := 0; r < len(s); r++ {
        if need[s[r]] > 0 { missing-- }
        need[s[r]]--
        for missing == 0 {
            if bl == -1 || r-l < br-bl { bl, br = l, r }
            need[s[l]]++
            if need[s[l]] > 0 { missing++ }
            l++
        }
    }
    if bl == -1 { return "" }
    return s[bl : br+1]
}

Longest repeating character replacement

// At most k changes. Window valid if (windowLen - maxFreq) <= k.
func characterReplacement(s string, k int) int {
    var c [26]int
    l, maxFreq, best := 0, 0, 0
    for r := 0; r < len(s); r++ {
        c[s[r]-'A']++
        if c[s[r]-'A'] > maxFreq { maxFreq = c[s[r]-'A'] }
        for r-l+1-maxFreq > k {
            c[s[l]-'A']--; l++
        }
        if r-l+1 > best { best = r-l+1 }
    }
    return best
}

Maximum in sliding window (monotonic deque)

func maxSlidingWindow(a []int, k int) []int {
    var dq []int           // stores indices, values decreasing
    res := make([]int, 0, len(a)-k+1)
    for i, v := range a {
        for len(dq) > 0 && a[dq[len(dq)-1]] < v { dq = dq[:len(dq)-1] }
        dq = append(dq, i)
        if dq[0] <= i-k { dq = dq[1:] }
        if i >= k-1 { res = append(res, a[dq[0]]) }
    }
    return res
}

When does sliding window work?

The condition must be monotonic: extending the window can only make the constraint stay valid or become invalid; shrinking can only restore validity. Without monotonicity, fall back to brute force or DP.

Anti-patterns — what NOT to say

  • "I'll recompute the window aggregate every time." That's O(n) per step → O(n²) overall. Maintain it incrementally.
  • Using a hash map for fixed-size ASCII windows. [26]int / [128]int is faster.
  • Trying sliding window when negatives invalidate monotonicity → fall back to prefix-sum + map.
  • Re-adding the entire window in map when a counter would do (Permutation in String).
  • Forgetting that delete(map, key) on freq=0 is required to keep "distinct count" cheap.

Senior interviewer follow-ups

ProbeWhat they want to hear
"Why is the inner while loop still O(n) total?"Each element enters and leaves the window once; l only moves forward.
"Sliding window vs two pointers — line in the sand?"Sliding window has a predicate on the window contents; two-pointer is more about pair-finding / partition.
"How do you do Sliding Window Maximum?"Monotonic deque: indices, values strictly decreasing; pop tail while smaller, pop head if out of window.
"How do you handle 'exactly K' when only ≤ K is easy?"solve(K) – solve(K-1).
"What if the array contains negatives and you want subarray sum = K?"Switch to prefix sum + count map.
"Stream version of longest substring without repeats?"Same algorithm; window indices grow, last-seen map is bounded by alphabet size.
