Heaps / Priority Queue

Anything with “k-th”, “top-K”, “merge k sorted”, “median in a stream”, or “shortest path” probably wants a heap.

Mental model

  • The trigger words: "k-th", "top-K", "smallest/largest so far", "merge K sorted", "median in stream", "shortest path", "reorganize".
  • Heap = priority queue with O(log n) push/pop and O(1) peek. Backed by a binary heap in an array (Go's container/heap).
  • Three high-leverage patterns:
    1. Heap of size K — keep the top-K with O(n log k).
    2. Two heaps — max-heap of lower half + min-heap of upper half for stream median.
    3. Lazy deletion — push fresh entries with timestamps/version, skip stale tops on pop.
  • Don't reach for a heap when bucket sort or counting works. Top-K-frequent in O(n) beats O(n log k) when the value range is bounded.
  • For Dijkstra, the heap stores (dist, node); skip the entry if it doesn't match the current best dist (lazy deletion).

The Go boilerplate (copy once, customize)

import "container/heap"

type IntHeap []int                            // min-heap
func (h IntHeap) Len() int            { return len(h) }
func (h IntHeap) Less(i, j int) bool  { return h[i] < h[j] }
func (h IntHeap) Swap(i, j int)       { h[i], h[j] = h[j], h[i] }
func (h *IntHeap) Push(x any)         { *h = append(*h, x.(int)) }
func (h *IntHeap) Pop() any           { old := *h; n := len(old); x := old[n-1]; *h = old[:n-1]; return x }

h := &IntHeap{}
heap.Init(h)
heap.Push(h, 5)
top := (*h)[0]
heap.Pop(h)
Flip Less for max-heap. For custom items, store structs (e.g. type Item struct{ Val, Pri int }).

K-th largest (min-heap of size k)

func kthLargest(a []int, k int) int {
    h := &IntHeap{}
    for _, v := range a {
        heap.Push(h, v)
        if h.Len() > k { heap.Pop(h) }
    }
    return (*h)[0]
}

Top-K Frequent (bucket sort = O(n))

func topKFrequent(a []int, k int) []int {
    freq := map[int]int{}
    for _, v := range a { freq[v]++ }
    buckets := make([][]int, len(a)+1)
    for v, f := range freq { buckets[f] = append(buckets[f], v) }
    res := make([]int, 0, k)
    for i := len(buckets) - 1; i >= 0 && len(res) < k; i-- {
        res = append(res, buckets[i]...)
    }
    return res[:k]
}

Merge K sorted lists (heap)

type Node struct{ Val int; Next *Node }
type NodeHeap []*Node
func (h NodeHeap) Len() int            { return len(h) }
func (h NodeHeap) Less(i, j int) bool  { return h[i].Val < h[j].Val }
func (h NodeHeap) Swap(i, j int)       { h[i], h[j] = h[j], h[i] }
func (h *NodeHeap) Push(x any)         { *h = append(*h, x.(*Node)) }
func (h *NodeHeap) Pop() any           { old := *h; n := len(old); x := old[n-1]; *h = old[:n-1]; return x }

func mergeK(lists []*Node) *Node {
    h := &NodeHeap{}
    for _, l := range lists { if l != nil { heap.Push(h, l) } }
    dummy := &Node{}; tail := dummy
    for h.Len() > 0 {
        n := heap.Pop(h).(*Node)
        tail.Next = n; tail = n
        if n.Next != nil { heap.Push(h, n.Next) }
    }
    return dummy.Next
}

Find Median from Data Stream (two heaps)

// max-heap (lower half) + min-heap (upper half). Sizes differ by at most 1.
type MedianFinder struct{ lo MaxHeap; hi MinHeap }
func (mf *MedianFinder) Add(x int) {
    if mf.lo.Len() == 0 || x <= mf.lo.Peek() { heap.Push(&mf.lo, x) } else { heap.Push(&mf.hi, x) }
    // rebalance
    if mf.lo.Len() > mf.hi.Len()+1 { heap.Push(&mf.hi, heap.Pop(&mf.lo)) }
    if mf.hi.Len() > mf.lo.Len()   { heap.Push(&mf.lo, heap.Pop(&mf.hi)) }
}
func (mf *MedianFinder) Median() float64 {
    if mf.lo.Len() > mf.hi.Len() { return float64(mf.lo.Peek()) }
    return (float64(mf.lo.Peek()) + float64(mf.hi.Peek())) / 2
}

Dijkstra (heap)

type Edge struct{ To, W int }
type PQItem struct{ Node, Dist int }
type PQ []PQItem
// implement heap.Interface with Less by Dist

func dijkstra(n int, src int, g [][]Edge) []int {
    const INF = int(1e18)
    dist := make([]int, n); for i := range dist { dist[i] = INF }
    dist[src] = 0
    h := &PQ{{src, 0}}
    for h.Len() > 0 {
        it := heap.Pop(h).(PQItem)
        if it.Dist > dist[it.Node] { continue }   // stale entry
        for _, e := range g[it.Node] {
            if nd := it.Dist + e.W; nd < dist[e.To] {
                dist[e.To] = nd
                heap.Push(h, PQItem{e.To, nd})
            }
        }
    }
    return dist
}

Anti-patterns — what NOT to say

  • "Sort the array, return a[k-1]." Works, but O(n log n) when O(n log k) or O(n) avg is on the table.
  • "Push everything into the heap." Heap of size K beats heap of size N when K ≪ N.
  • Forgetting to discard stale entries in lazy-deletion patterns (Dijkstra, sliding median).
  • Using container/list or hand-rolled heap when container/heap is the tool.
  • Using a max-heap when a min-heap was needed (or vice versa) — flip Less.

Senior interviewer follow-ups

ProbeWhat they want to hear
"Heap of K vs quickselect for K-th largest?"Heap is O(n log k), online (handles streams). Quickselect is O(n) avg, offline, in-place.
"Why two heaps for stream median?"Lower half = max-heap, upper half = min-heap; rebalance to keep sizes within 1; median is the top(s).
"How do you delete an arbitrary element from a heap?"Standard heap doesn't support it. Either lazy deletion (mark + skip on pop) or use an indexed heap (map element → heap index).
"Dijkstra with heap on a dense graph?"O((V+E) log V). For dense graphs (E ≈ V²), an O(V²) array-based Dijkstra can win.
"Merge K sorted streams of unknown size?"Heap of one element per stream; pop and re-push next element of that stream.
"Reorganize string with constraint of distance ≥ d?"Generalized Task Scheduler — max-heap + cooldown queue of size d.
"Why do you push (dist, node) with possibly stale dist in Dijkstra?"Cheaper than a decrease-key operation; on pop, skip if dist is stale.
