The Complete LeetCode Trick Book

Every battle-tested heuristic, pattern signal, and code micro-trick that turns a 45-minute struggle into a 10-minute solve. Skim before any interview.

0. The 5-step universal solving framework

  1. Clarify — inputs, outputs, ranges, duplicates, negatives, empty, sortedness, in-place allowed?
  2. Examples — 1 normal + 1 edge (empty/1-el/all-equal/sorted-reverse).
  3. Brute force — say it out loud with complexity. Often unlocks the optimization.
  4. Optimize — apply a pattern (table below). State new complexity before coding.
  5. Code → trace → test edges.

3. Universal code micro-tricks (Go)

Index arithmetic

// Safe mid (no overflow):
m := lo + (hi-lo)/2

// 2D ↔ 1D index conversion:
idx := r*cols + c
r, c := idx/cols, idx%cols

// Iterate 4 / 8 neighbors:
dr := []int{-1, 1, 0, 0}
dc := []int{0, 0, -1, 1}
for k := 0; k < 4; k++ {
    nr, nc := r+dr[k], c+dc[k]
    if nr >= 0 && nr < R && nc >= 0 && nc < C { /* ... */ }
}

// Diagonals on a grid: i+j is constant on / diagonals; i-j on \ diagonals.

Counters without map (ASCII / lowercase)

var c [26]int                  // for 'a'..'z'  → faster than map[byte]int
c[s[i]-'a']++
// Compare anagram signatures with == on fixed-size arrays — no DeepEqual.

“Mark visited” without extra space

// In a positive-int array, negate to mark "seen".
for _, x := range a {
    i := abs(x) - 1
    if a[i] > 0 { a[i] = -a[i] }   // i has been seen
}
// For grids, overwrite with a sentinel char ('#') instead of a visited[][] grid.

Sentinel & dummy nodes

// Linked-list problems: always start with a dummy head.
dummy := &ListNode{Next: head}
// Stack problems: push a sentinel to flush the monotonic stack at the end.
heights = append(heights, 0)

Sort by multi-key

sort.Slice(items, func(i, j int) bool {
    if items[i].Key != items[j].Key { return items[i].Key < items[j].Key }
    return items[i].Val > items[j].Val      // tie-break
})

Min/max for ints (Go 1.21+)

x := min(a, b)
y := max(a, b, c)
// Pre-1.21: write tiny helpers or use a generic Ordered constraint.

Heap — copy this once and re-use

type IntHeap []int
func (h IntHeap) Len() int            { return len(h) }
func (h IntHeap) Less(i, j int) bool  { return h[i] < h[j] }   // min-heap
func (h IntHeap) Swap(i, j int)       { h[i], h[j] = h[j], h[i] }
func (h *IntHeap) Push(x any)         { *h = append(*h, x.(int)) }
func (h *IntHeap) Pop() any           { old := *h; n := len(old); x := old[n-1]; *h = old[:n-1]; return x }
// Use:  h := &IntHeap{}; heap.Init(h); heap.Push(h, 5); heap.Pop(h).(int)

4. Pattern-specific tricks

Arrays / prefix

  • “Subarray sum equals K” → map[prefixSum]count; at index i, answer += map[prefix - K].
  • Product except self → two passes: left-product then right-product, both into the same output array.
  • Difference array for range updates + 1 final pass: add v at l, subtract v at r+1, then prefix-sum.
  • Boyer–Moore majority — candidate + count. O(n) / O(1).
  • 2D prefix sum: P[i][j] = a[i-1][j-1] + P[i-1][j] + P[i][j-1] - P[i-1][j-1].

Two pointers

  • “Squares of sorted array” → fill the result from the back, choosing the larger of |a[l]| and |a[r]|.
  • For k-Sum: sort, then nested two-pointer; skip duplicates after a hit for uniqueness.
  • “In-place compaction” → write pointer + read pointer; condition controls when to advance write.

Sliding window

  • “Exactly K distinct” = “at most K” − “at most K-1”. Same template, two calls.
  • Use a fixed-size counter array for ASCII, integer distinct counter is faster than rechecking the map.
  • For fixed-window problems, add new element first, then drop the leaving element, then record answer.

Binary search

  • Phrase as monotone predicate P(x) over the answer space. The search reduces to “lowest true”.
  • Use half-open [lo, hi); updates: lo = m+1 or hi = m. No infinite loops, ever.
  • Rotated array → check which half is sorted using a[lo] <= a[m].
  • Floating-point: iterate 60–100 times instead of comparing for equality.

Linked lists

  • Always use a dummy head; simplifies head-deletion and merge logic.
  • Floyd's cycle: after meeting, reset one pointer to head, advance both by 1; meet point = cycle start.
  • Reverse, swap pairs, reorder list → all variations of the 3-pointer reverse.
  • Length difference trick (two lists): advance the longer by Δ first, then walk together.

Stacks & queues

  • Monotonic stack for next-greater/smaller, histograms, daily temperatures. Push indices, not values.
  • Append a sentinel 0 at the end of histograms to flush remaining stack entries.
  • “Largest rectangle in histogram” → monotonic increasing stack of indices.
  • Two stacks = queue (amortized O(1)). Stack with min/max → store (value, runningMin) pairs.

Hashing

  • Sliding count map: when count hits 0, delete the key so len(map) equals distinct.
  • For grouping-by-shape (anagrams, shifted strings, isomorphic) → derive a canonical key.
  • For 2-sum style → store complement → index, single pass.
  • Use a struct or [N]int as a map key — Go allows comparable composite keys.

Heaps

  • k-th largest → min-heap of size k. k-th smallest → max-heap of size k.
  • Two heaps for running median: max-heap (lower half) + min-heap (upper half), rebalance after each insert.
  • Merge k sorted lists → heap of (val, listIdx, nodeIdx), pop & push next.

Trees

  • Recursion returns the useful local quantity (depth, sum, max-path-through-node) — not just the final answer. Update a closure-captured global.
  • BST → in-order traversal is sorted; use iterative stack for k-th smallest.
  • LCA: post-order recursion; node returns itself if it equals p or q; if both children non-nil → current.
  • Serialize with pre-order + null markers; deserialize with a queue/index.
  • Tree DP: at each node, return ≥1 number to its parent (e.g. (robThis, skipThis) for House Robber III).

Tries

  • Array children [26]*Node for lowercase only; map for arbitrary alphabets.
  • Mark isWord flag at end nodes; store extra metadata (count, indexes) for autocomplete.
  • XOR-max-pair → binary trie of bits, greedy go-opposite-bit at each level.

Graphs

  • Build adjacency list once: adj := make([][]int, n).
  • BFS for unweighted shortest path; record parent for path reconstruction.
  • Multi-source BFS: push all sources at step 0 — used for “nearest 0”, “rotting oranges”.
  • 0/1-BFS (edges of weight 0 or 1) → deque; push-front for weight-0 edges, push-back for weight-1.
  • Dijkstra: lazy deletion (push duplicates, skip when popped distance > known) is simpler than decrease-key.
  • Detect cycle in directed graph → DFS with 3 colors (white/gray/black) or Kahn's in-degree.
  • Detect cycle in undirected → DFS tracking parent, or Union-Find while iterating edges.
  • Bipartite check → BFS 2-coloring.
  • Bridges & articulation points → Tarjan's disc/low arrays.
  • Strongly connected components → Tarjan or Kosaraju (two DFS passes on the original then transpose).

Backtracking

  • Template: choose → recurse → un-choose. Modify shared slice/map in place; copy on save.
  • Append a copy of the current path before pushing into results: res = append(res, append([]int(nil), path...)).
  • Order matters → permutations (used[]). Order doesn't → combinations (pass a start index).
  • Prune early: sort first, break when sum exceeds target, skip duplicates with i > start && nums[i]==nums[i-1].
  • N-Queens → 3 sets: cols, diag1 (r+c), diag2 (r-c+n).
  • Sudoku → bitmask row/col/box, recurse on the most-constrained cell first.

Greedy

  • Try sorting by one key (start, end, ratio). 90% of greedy problems collapse after the right sort.
  • Prove correctness with an exchange argument: swap to “optimal” order without losing value.
  • Interval scheduling → sort by end time, pick first that doesn't overlap.
  • Jump Game → keep maxReach; if i > maxReach, impossible.
  • Gas station → if total < 0 impossible; otherwise the answer is right after the last point where running sum went negative.

Dynamic programming

  • Identify state first (the smallest tuple that fully describes a sub-problem). Then transitions.
  • Top-down (memo) is easier to derive; bottom-up gives space-rolling.
  • 1D DP with dp[i] based on a few previous → roll to O(1) space.
  • 2D DP with dp[i][j] only depending on previous row → roll to two rows.
  • 0/1 knapsack: iterate capacity from high to low (1D).
  • Unbounded knapsack: iterate capacity from low to high.
  • LIS in O(n log n): maintain tails, binary-search insertion point.
  • Interval DP (matrix chain, burst balloons): outer loop = length; inner = left; split = k in (l, r).
  • Digit DP: recurse on (pos, tight, leadingZero, state); memo on (pos, state) when not tight.
  • Bitmask DP for n ≤ 20: state is a subset; transitions add one element.

Bit manipulation

x & (x-1)        // clears lowest set bit
x & -x           // isolates lowest set bit
bits.OnesCount(uint(x))      // popcount
bits.TrailingZeros(uint(x))  // index of lowest set bit
x ^ y            // diff; XOR all numbers to find the single unique one
// Enumerate every submask of mask:
for sub := mask; sub > 0; sub = (sub-1) & mask { /* ... */ }
  • Single number among pairs → XOR everything.
  • Two unique numbers among pairs → XOR all, isolate any set bit, split into 2 groups, XOR each.
  • Subset enumeration → iterate 0..(1<<n)-1.

Math & number theory

  • gcd(a, b) = gcd(b, a%b); lcm = a / gcd * b (multiply after division to avoid overflow).
  • Modular exponentiation in O(log n) via square-and-multiply.
  • Sieve of Eratosthenes for primes ≤ 10⁷.
  • Fast inverse mod p (prime): pow(a, p-2, p).
  • Reservoir sampling for k items from a stream of unknown length.

5. Edge-case checklist (run mentally before submitting)

  • Empty input ("", [], nil)
  • Single element / two equal elements
  • All elements equal / strictly increasing / strictly decreasing
  • Negative numbers, zero, integer overflow
  • Maximum/minimum allowed input size (does your O(n²) fit?)
  • Duplicates in input (esp. for k-Sum, BST insertion)
  • Disconnected graph / cycle / self-loop
  • Linked list with 0, 1, 2 nodes
  • Recursion depth (stack overflow on n = 10⁵ linked list reverse)
  • Off-by-one on inclusive / exclusive bounds

7. Anti-patterns that lose interviews

  • Coding before stating an approach and complexity.
  • Using map when a 26-element array suffices.
  • Recursing without explaining base case / memo size.
  • Re-allocating in a hot loop (string += in a tight loop).
  • Mutating input without warning the interviewer.
  • Not closing the loop: forgetting to update min/max, missing the last window, off-by-one on lower-bound binary search.
  • Ignoring overflow — use int64 when sums can exceed 2³¹.

8. Time-boxed interview routine

  • 0–2 min: restate problem, ask 3 clarifying questions, list assumptions.
  • 2–5 min: walk through 2 examples (1 normal, 1 edge).
  • 5–10 min: brute force → optimization → final complexity. Get explicit “sounds good”.
  • 10–30 min: code, narrate each block, name variables meaningfully.
  • 30–40 min: dry-run on the small example, then an edge case. Fix any bugs.
  • 40–45 min: discuss follow-ups (scale, concurrency, memory, streaming).
