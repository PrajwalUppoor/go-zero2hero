Two Pointers

When the array is sorted (or can be), two pointers crush O(n²) brute force into O(n). Use it for pair sums, partitions, palindromes, in-place compaction.

Mental model

  • Three flavors:
    1. Opposite ends — converging on a sorted array (pair sum, palindrome, container).
    2. Same direction, fast/slow — write/read pointers for in-place compaction; cycle detection on linked lists.
    3. Two arrays in lockstep — merging, comparing.
  • The win condition: each pointer moves at most n times, so the loop is O(n) instead of O(n²).
  • The trigger: monotonicity. Moving a pointer should only ever tighten or loosen a constraint, never both.
  • If the array isn't sorted, sort it first only if indices don't matter in the answer; otherwise reach for hashing.

Pattern A — Opposite ends (sorted array)

func twoSumSorted(a []int, target int) (int, int) {
    l, r := 0, len(a)-1
    for l < r {
        s := a[l] + a[r]
        switch {
        case s == target: return l, r
        case s < target:  l++
        default:          r--
        }
    }
    return -1, -1
}

Pattern B — Same direction (write index)

// Remove duplicates from sorted array, in-place.
func removeDuplicates(a []int) int {
    w := 0
    for r := 0; r < len(a); r++ {
        if w == 0 || a[r] != a[w-1] {
            a[w] = a[r]; w++
        }
    }
    return w
}

Pattern C — Three sum

func threeSum(nums []int) [][]int {
    sort.Ints(nums)
    var res [][]int
    for i := 0; i < len(nums)-2; i++ {
        if i > 0 && nums[i] == nums[i-1] { continue }
        l, r := i+1, len(nums)-1
        for l < r {
            s := nums[i] + nums[l] + nums[r]
            switch {
            case s == 0:
                res = append(res, []int{nums[i], nums[l], nums[r]})
                l++; r--
                for l < r && nums[l] == nums[l-1] { l++ }
                for l < r && nums[r] == nums[r+1] { r-- }
            case s < 0: l++
            default:    r--
            }
        }
    }
    return res
}

Pattern D — Container with most water / trap rain

func maxArea(h []int) int {
    l, r, best := 0, len(h)-1, 0
    for l < r {
        area := min(h[l], h[r]) * (r - l)
        if area > best { best = area }
        if h[l] < h[r] { l++ } else { r-- }
    }
    return best
}

func trap(h []int) int {
    l, r := 0, len(h)-1
    lm, rm, ans := 0, 0, 0
    for l < r {
        if h[l] < h[r] {
            if h[l] > lm { lm = h[l] } else { ans += lm - h[l] }
            l++
        } else {
            if h[r] > rm { rm = h[r] } else { ans += rm - h[r] }
            r--
        }
    }
    return ans
}

When NOT to use two pointers

  • Unsorted data with no derivable monotonic invariant — try hashing or sorting first.
  • You need contiguous subarrays with a sum constraint that can grow either direction → use sliding window or prefix sums.
  • Sum can be negative AND target is fixed → prefix-sum + hashmap, not two pointers.
  • You need indices in the original (unsorted) array → can't sort destructively.

Anti-patterns — what NOT to say

  • "I'll use two pointers on the unsorted array." Won't work without monotonicity — you'll skip valid pairs.
  • Using two pointers when negatives + arbitrary target → reach for hashmap.
  • Forgetting to skip duplicates after a hit in 3-Sum / 4-Sum — the answer set will contain duplicates.
  • Advancing both pointers on equal-sum case in 3-Sum without skipping duplicate values.
  • Sorting (and losing original indices) when the problem expects original indices.

Senior interviewer follow-ups

ProbeWhat they want to hear
"Three Sum without sorting?"Hashmap-based: for each pair, look up complement. Same time, more space, same dedup pain.
"Why does the converging-pointer move the smaller side in Container With Most Water?"Moving the larger side can only shrink width and potentially shrink height — never improves.
"Trap Rain Water with O(1) space — how?"Two-pointer; whichever side has the smaller max can be filled deterministically.
"How does Dutch flag handle 4 colors?"Generalizes to k buckets; consider counting sort or k-1 passes of partition.
"Two pointers vs sliding window — when is which?"Two pointers usually move toward each other or have a write-vs-read role; sliding window is a single direction with grow/shrink under a predicate.

Pitfalls

  • Off-by-one when checking l < r vs l <= r — both are correct in different contexts; pick one and own it.
  • Forgetting that "skip duplicates" must use a previous-value comparison, not previous-index.
  • For palindromes with mixed case / punctuation, normalize first or skip in-place — don't mutate the input.

Self-check

  • I can write Two Sum II from memory in O(n) / O(1).
  • I can write Three Sum and handle duplicate skipping correctly.
  • I can solve Trapping Rain Water in O(n) / O(1) with two pointers.
  • I can implement Dutch flag (Sort Colors) in one pass.
  • I can do in-place removal/move-zeroes with read/write pointers.
  • I can merge two sorted arrays in place (write from the back).
  • I know when two pointers fails (negatives + sum, or unsorted).
  • I can solve Squares of a Sorted Array in O(n).
  • I can recognize the "skip duplicates" requirement and code it.
  • I can explain why opposite-end converging moves the smaller side.
