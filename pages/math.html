Math & Number Theory

The classics that get smuggled into coding problems: gcd, modular exponentiation, primes, combinatorics, reservoir sampling.

Mental model

  • Five recurring tools:
    1. Euclidean gcd — also gives modular inverse via the extended version.
    2. Fast exponentiation — O(log e), works for integers, matrices, and modular exponents.
    3. Sieve of Eratosthenes — primes up to N in O(N log log N).
    4. Modular arithmetic with a prime modulus — for combinatorics, hashing, RSA-flavored problems.
    5. Reservoir / weighted sampling — when the dataset is unbounded.
  • Watch overflow: Go's int is 64-bit on 64-bit hosts, but multiplications can still overflow. Reduce mod after every multiply.
  • For huge numbers: math/big. For float precision: math/big.Float or math.FMA.
  • For randomness: math/rand/v2 in 1.22+ has cleaner API. Seed once, share state.

gcd & lcm

func gcd(a, b int) int { for b != 0 { a, b = b, a%b }; return a }
func lcm(a, b int) int { return a / gcd(a, b) * b }

Modular exponentiation

func powMod(a, e, m int) int {
    r := 1 % m
    a %= m
    for e > 0 {
        if e & 1 == 1 { r = r * a % m }
        a = a * a % m
        e >>= 1
    }
    return r
}

Sieve of Eratosthenes

func sieve(n int) []int {
    is := make([]bool, n+1)
    for i := 2; i*i <= n; i++ {
        if !is[i] {
            for j := i * i; j <= n; j += i { is[j] = true }
        }
    }
    var primes []int
    for i := 2; i <= n; i++ { if !is[i] { primes = append(primes, i) } }
    return primes
}

Combinatorics — nCr mod p

// Precompute factorials and inverse factorials mod p (prime).
const MOD = 1_000_000_007
var fact, inv []int

func precompute(n int) {
    fact = make([]int, n+1); inv = make([]int, n+1)
    fact[0] = 1
    for i := 1; i <= n; i++ { fact[i] = fact[i-1] * i % MOD }
    inv[n] = powMod(fact[n], MOD-2, MOD)
    for i := n - 1; i >= 0; i-- { inv[i] = inv[i+1] * (i + 1) % MOD }
}

func nCr(n, r int) int {
    if r < 0 || r > n { return 0 }
    return fact[n] * inv[r] % MOD * inv[n-r] % MOD
}

Reservoir sampling — k items from a stream

// k = 1; pick element i with probability 1/i.
func sample(stream []int) int {
    chosen, seen := 0, 0
    for _, v := range stream {
        seen++
        if rand.Intn(seen) == 0 { chosen = v }
    }
    return chosen
}

Random pick with weights

// Build prefix sum of weights; binary search a random in [1, total].
type Picker struct{ pre []int }
func New(w []int) *Picker {
    p := &Picker{pre: make([]int, len(w))}
    s := 0
    for i, x := range w { s += x; p.pre[i] = s }
    return p
}
func (p *Picker) Pick() int {
    r := rand.Intn(p.pre[len(p.pre)-1]) + 1
    return sort.SearchInts(p.pre, r)
}

Overflow & big integers

  • Go's int is 64-bit on 64-bit OSes. Multiplications can still overflow → use math/big for arbitrary precision.
  • For modulus problems, keep intermediate values within int64; reduce after every multiply.

Anti-patterns — what NOT to say

  • Computing a^e with naive multiplication when e > 30 — overflow + slow.
  • Using math.Pow for integer exponents — float precision loss.
  • Modular inverse via brute-force when the modulus is prime — use Fermat's little theorem.
  • Building all primes when the trial-division of a single number suffices.
  • Generating random numbers without seeding (or seeding repeatedly) — both are bugs.
  • Using rand.Intn in concurrent code without synchronization (math/rand v1).

Senior interviewer follow-ups

ProbeWhat they want to hear
"Why does fast exponentiation work?"Square the base when the bit is 0; multiply into the answer when the bit is 1. Each iteration halves the exponent.
"Modular inverse — Fermat's little vs extended Euclid?"Fermat: a^(p-2) mod p if p is prime. Extended Euclid works for any coprime modulus.
"Reservoir sampling correctness?"At step i, replace with probability 1/i; induction shows uniform distribution at the end.
"Random pick with weights — why prefix sum?"Map a uniform random to the cumulative distribution; binary-search for the first prefix ≥ random.
"Why is sieve O(n log log n)?"Sum over primes p ≤ n of n/p ≈ n log log n by Mertens' theorem.
"Detecting overflow in Go?"No language-level overflow check. Use math/big or guard manually with if a > max/b.
