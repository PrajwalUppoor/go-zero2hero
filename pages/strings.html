Strings

String hashing, KMP, Z-function, palindromes, sliding windows over chars, anagrams. Go's string is read-only UTF-8 — know when to convert to []byte or []rune.

Mental model

  • A Go string is a read-only slice of UTF-8 bytes. Indexing returns a byte; iteration with range returns runes (decoded code points).
  • Three techniques cover most problems: counter array (anagrams), sliding window (substring conditions), and rolling-hash / KMP (pattern matching).
  • If the alphabet is fixed and small (lowercase ASCII), prefer [26]int arrays over maps — faster, copy-cheap, hashable.
  • For palindromes, expand-around-center is the bread-and-butter. Manacher only when O(n) is required.
  • For repeated string searches in a fixed text, build a suffix array or suffix automaton; for needle/haystack, KMP / Z is plenty.

Go string fundamentals

  • len(s) = bytes. To count code points: utf8.RuneCountInString(s) or convert to []rune.
  • for i, r := range s iterates runes; i is the byte offset.
  • Mutate via []byte or []rune, then convert back.
  • Use strings.Builder for concatenation.

Anagram check (counter)

func isAnagram(s, t string) bool {
    if len(s) != len(t) { return false }
    var c [26]int
    for i := 0; i < len(s); i++ { c[s[i]-'a']++; c[t[i]-'a']-- }
    for _, v := range c { if v != 0 { return false } }
    return true
}

Group anagrams

func groupAnagrams(ws []string) [][]string {
    g := map[[26]int][]string{}
    for _, w := range ws {
        var k [26]int
        for i := 0; i < len(w); i++ { k[w[i]-'a']++ }
        g[k] = append(g[k], w)
    }
    out := make([][]string, 0, len(g))
    for _, v := range g { out = append(out, v) }
    return out
}

Longest palindromic substring (expand around center)

func longestPalindrome(s string) string {
    if s == "" { return "" }
    bestL, bestR := 0, 0
    expand := func(l, r int) (int, int) {
        for l >= 0 && r < len(s) && s[l] == s[r] { l--; r++ }
        return l + 1, r - 1
    }
    for i := 0; i < len(s); i++ {
        l1, r1 := expand(i, i)
        l2, r2 := expand(i, i+1)
        if r1-l1 > bestR-bestL { bestL, bestR = l1, r1 }
        if r2-l2 > bestR-bestL { bestL, bestR = l2, r2 }
    }
    return s[bestL : bestR+1]
}

KMP — pattern matching in O(n+m)

func kmpFail(p string) []int {
    f := make([]int, len(p))
    k := 0
    for i := 1; i < len(p); i++ {
        for k > 0 && p[k] != p[i] { k = f[k-1] }
        if p[k] == p[i] { k++ }
        f[i] = k
    }
    return f
}
func kmpSearch(t, p string) int {
    if p == "" { return 0 }
    f := kmpFail(p); k := 0
    for i := 0; i < len(t); i++ {
        for k > 0 && p[k] != t[i] { k = f[k-1] }
        if p[k] == t[i] { k++ }
        if k == len(p) { return i - k + 1 }
    }
    return -1
}

Anti-patterns — what NOT to say

  • "I'll convert to []rune just to be safe." That's an O(n) allocation. For ASCII, work with bytes.
  • String concatenation in a loop with +=. Use strings.Builder with a sized Grow hint.
  • Hand-rolling regex when strings.IndexByte / strings.Cut are clearer and faster.
  • Using a hashmap of byte counts when a fixed array [26]int is faster, hashable, and copyable.
  • Single-mod rolling hash for adversarial inputs — collisions are exploitable. Use double hashing.

Senior interviewer follow-ups

ProbeWhat they want to hear
"Why is len(s) not the same as character count for non-ASCII?"Strings are UTF-8 byte slices; multi-byte runes count once but occupy 2–4 bytes.
"How would you make Longest Palindromic Substring O(n)?"Manacher's algorithm — keep expanding around centers using mirror symmetry.
"Why is the failure function in KMP suffix-prefix table?"It tells you the longest proper prefix of p[0..i] that is also a suffix — when a mismatch happens, you can skip to that prefix.
"Two strings; same characters, different order — how do you compare in O(n)?"Counter array (or sort with O(n log n)).
"How to find all duplicate substrings of length k?"Rolling hash with double mod, storing seen hashes.
"Concat 1 million strings — how?"strings.Builder with Grow(totalSize) precomputed.
"Detect anagram across very long strings?"Counter compare; or sort each and compare; or 26-letter hash.

Pitfalls

  • Indexing a string returns a byte, not a rune. For multibyte text, iterate with range or use utf8.DecodeRuneInString.
  • []rune(s) allocates O(n). For ASCII-only, work with bytes.
  • Using strings.Index in a hot loop instead of one-shot KMP / Z when scanning many patterns.
  • Forgetting that strings.Builder can't be copied — pass by pointer.
  • Rolling-hash collisions — always confirm with a direct compare on suspected match.
