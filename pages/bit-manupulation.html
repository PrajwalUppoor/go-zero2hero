Bit Manipulation

Constant-time tricks, subset enumeration, bitmask DP, and XOR magic for "find the one different".

Mental model

  • Trigger words: "find the one different / unique", "all subsets", "without using +", "count bits", "constant-time math", "n ≤ 20" (bitmask DP).
  • Three high-leverage techniques:
    1. XOR magic — pairs cancel; the leftover is the unpaired element.
    2. Subset enumeration — for mask in 0..2^n; or submasks via sub = (sub-1) & mask.
    3. Lowest-set-bit isolation — x & -x, the basis of Fenwick trees and many "find a difference" tricks.
  • Use math/bits: OnesCount, TrailingZeros, LeadingZeros, Len, Reverse. They compile to single instructions.
  • Beware sign extension: right-shifting a signed int preserves sign in Go. Cast to uint for logical shift.

The 12 must-know tricks

x & 1                 // lowest bit (parity)
x >> 1                // divide by 2
x << k                // multiply by 2^k
x | (1 << k)         // set bit k
x & ^(1 << k)         // clear bit k
x ^ (1 << k)          // toggle bit k
(x >> k) & 1          // read bit k
x & (x-1)             // clear lowest set bit  → loop runs popcount times
x & -x                // isolate lowest set bit  (uses two's complement)
x ^ y                 // toggle bits where different (find single duplicate)
^x                    // bitwise NOT
bits.OnesCount(uint(x))      // popcount
bits.TrailingZeros(uint(x))  // index of lowest set bit
bits.LeadingZeros(uint(x))   // 64 - msb position

Classic problems

Single Number (everyone twice, one once)

func singleNumber(a []int) int {
    x := 0
    for _, v := range a { x ^= v }
    return x
}

Two non-duplicates among pairs

// XOR all → x = a^b. Pick any set bit; split numbers into two groups by that bit.
func singleNumberIII(a []int) [2]int {
    x := 0
    for _, v := range a { x ^= v }
    bit := x & -x
    var p, q int
    for _, v := range a { if v & bit == 0 { p ^= v } else { q ^= v } }
    return [2]int{p, q}
}

Single Number II (everyone three times, one once)

// Sum each bit modulo 3.
func singleNumberII(a []int) int {
    ans := 0
    for b := 0; b < 32; b++ {
        s := 0
        for _, v := range a { s += (v >> b) & 1 }
        if s%3 != 0 { ans |= 1 << b }
    }
    return ans
}

Counting bits 0..n in O(n)

// bits[i] = bits[i >> 1] + (i & 1)
func countBits(n int) []int {
    res := make([]int, n+1)
    for i := 1; i <= n; i++ { res[i] = res[i>>1] + (i & 1) }
    return res
}

Subsets via bitmask

func subsets(a []int) [][]int {
    n := len(a)
    res := make([][]int, 0, 1<<n)
    for mask := 0; mask < 1<<n; mask++ {
        var sub []int
        for i := 0; i < n; i++ {
            if mask & (1 << i) != 0 { sub = append(sub, a[i]) }
        }
        res = append(res, sub)
    }
    return res
}

Iterate every submask of a mask

for sub := mask; sub > 0; sub = (sub - 1) & mask {
    // process sub
}
// Total work over all masks is O(3^n).

Power of two / four

isPow2  := x > 0 && x & (x-1) == 0
isPow4  := isPow2 && (x & 0x55555555) != 0

Swap without temp (don't do this in code review)

a ^= b; b ^= a; a ^= b   // works only if a and b are distinct memory locations

Gotchas in Go

  • Right-shift on signed int is arithmetic (preserves sign); cast to uint for logical shift.
  • Shift count must be unsigned or a typed constant (since Go 1.13 either signed or unsigned is OK at runtime).
  • Bit count uses math/bits; it compiles to a single instruction on most CPUs.

Anti-patterns — what NOT to say

  • Counting bits with a manual loop when bits.OnesCount is one instruction.
  • Mixing signed and unsigned shifts inadvertently.
  • The XOR-swap "trick" in production code — undefined when both refs alias.
  • Iterating 2^n subsets for n > 20 — won't finish.
  • Using bit tricks where readable code suffices — code review will scold.

Senior interviewer follow-ups

ProbeWhat they want to hear
"Why does x & -x isolate the lowest set bit?"Two's complement: -x = ~x + 1; AND keeps only the bit that flipped from 0 to 1.
"Sum without +?"XOR is sum without carry; (a & b) << 1 is the carry; iterate until carry is 0.
"Maximum XOR of two numbers — why a binary trie?"For each candidate, greedily walk the trie picking the opposite bit when present.
"Counting bits in O(n)?"Recurrence on i >> 1 — the bit count is the count of half plus the lowest bit.
"Bitmask DP space — when is it feasible?"n ≤ ~20 (4M states). For n ≈ 30 we need profile + state compression.
