The 8-week roadmap

If you only have 1–2 weeks, jump straight to the last-minute cheatsheet and the flashcards.

Phase 1 — Go Language (Weeks 1–2)

  • Fundamentals: types, slices, maps, structs, methods, interfaces, errors
  • Advanced: generics, reflection, escape analysis, GC, profiling
  • Concurrency: goroutines, channels, sync, context, errgroup, patterns
  • Stdlib & idioms: net/http, encoding/json, testing, table tests

Phase 2 — Coding Patterns (Weeks 3–6)

  • Arrays, strings, two pointers, sliding window
  • Binary search, sorting, intervals, math, bits
  • Linked lists, stacks & queues, hashing, heaps
  • Trees, tries, graphs
  • Backtracking, greedy, DP (1D/2D, knapsack, LIS, intervals, bitmask)

Phase 3 — System Design (Weeks 7–8)

  • Fundamentals: latency, throughput, CAP, PACELC, consistency
  • Building blocks: LB, gateway, CDN, cache, queue, search
  • Databases · caching · messaging · microservices
  • Case studies: URL shortener, news feed, chat, rate limiter, search, distributed cache

Daily routine (the “blind” schedule)

  • Morning (45m) — read one pattern page; type the templates by hand.
  • Midday (60–90m) — solve 2–3 problems on that pattern.
  • Evening (20m) — review flashcards + 1 system-design building block.
  • Weekend — one mock system design + one mock coding round (45m + 45m).
